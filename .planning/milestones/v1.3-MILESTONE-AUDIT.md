### Prior Concern: Nostr Cold-Start Initialization — RESOLVED

The v1.3 19:00Z audit flagged a concern that `initNostr()` was not called at room page or scene page `onMount`, potentially causing wrong UI state on cold start. **Integration check confirms this is resolved:**

`+layout.svelte` line 47 calls `initNostr().then(() => subscribeToIncomingDMs())` in `onMount` — this fires on every page because the root layout's `onMount` executes before any child page's `onMount`. By the time the room or scene page runs `checkActiveRoom()`, NDK is already initialized.
