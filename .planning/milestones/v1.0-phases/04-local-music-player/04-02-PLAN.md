**src/lib/player/state.ts** — Svelte 5 runes-based player state:
- Define `PlayerTrack` interface: `path: string`, `title: string`, `artist: string`, `album: string`, `albumArtist?: string`, `trackNumber?: number`, `discNumber?: number`, `genre?: string`, `year?: number`, `durationSecs: number`
- Create a reactive state object using Svelte 5 module-level `$state`:
  ```typescript
  export const playerState = $state({
    currentTrack: null as PlayerTrack | null,
    isPlaying: false,
    currentTime: 0,
    duration: 0,
    volume: 1,
    isMuted: false,
    isLoading: false,
  });
  ```
- Export the state object. All components import and read/write this directly.

**src/lib/player/audio.ts** — HTML5 Audio engine:
- Create a module-scoped `HTMLAudioElement` (created lazily on first use, only in browser context)
- Implement `initAudio()`: creates the Audio element, attaches event listeners:
  - `timeupdate` → updates `playerState.currentTime`
  - `loadedmetadata` → updates `playerState.duration`, sets `playerState.isLoading = false`
  - `ended` → calls `playNext()` from queue
  - `play` → sets `playerState.isPlaying = true`
  - `pause` → sets `playerState.isPlaying = false`
  - `waiting` → sets `playerState.isLoading = true`
  - `canplay` → sets `playerState.isLoading = false`
- Implement `playTrack(track: PlayerTrack)`:
  - Ensure audio element exists (call initAudio if needed)
  - Use dynamic import: `const { convertFileSrc } = await import('@tauri-apps/api/core')`
  - Convert `track.path` to asset URL: `convertFileSrc(track.path)`
  - Set `audio.src = url`, call `audio.play()`
  - Update `playerState.currentTrack = track`
- Implement `pause()`, `resume()`, `togglePlayPause()`
- Implement `seek(seconds: number)`: set `audio.currentTime = seconds`
- Implement `setVolume(vol: number)`: set `audio.volume`, update `playerState.volume`. Clamp 0-1.
- Implement `toggleMute()`: store previous volume, set to 0 or restore
- Implement `getAudioElement()` for external access (e.g. keyboard shortcuts)
- All functions check `typeof window !== 'undefined'` before accessing Audio (SSR guard)

**src/lib/player/queue.ts** — Queue management:
- Create reactive queue state:
  ```typescript
  export const queueState = $state({
    tracks: [] as PlayerTrack[],
    currentIndex: -1,
    shuffled: false,
    repeatMode: 'none' as 'none' | 'all' | 'one',
  });
  ```
- Implement `setQueue(tracks: PlayerTrack[], startIndex?: number)`:
  - Sets queueState.tracks, sets currentIndex to startIndex ?? 0
  - Calls `playTrack(tracks[currentIndex])` from audio module
- Implement `addToQueue(track: PlayerTrack)`: appends to end of tracks array
- Implement `addToQueueNext(track: PlayerTrack)`: inserts after currentIndex
- Implement `playNext()`:
  - If repeatMode is 'one', replay current track
  - If at end and repeatMode is 'all', go to index 0
  - If at end and repeatMode is 'none', stop
  - Otherwise, increment index and play
- Implement `playPrevious()`:
  - If currentTime > 3s, seek to 0 (restart current track)
  - Otherwise go to previous index (clamped to 0)
- Implement `removeFromQueue(index: number)`
- Implement `clearQueue()`
- Implement `toggleShuffle()` and `toggleRepeat()`

- This is the public API: `import { playerState, playTrack, queueState, setQueue } from '$lib/player'`
