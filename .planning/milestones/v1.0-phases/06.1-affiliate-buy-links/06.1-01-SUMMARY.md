## Accomplishments
- Full `src/lib/affiliates/` module: types, config, construct, index
- `buildBuyLinks()` always returns exactly 5 BuyLinks (Bandcamp, Amazon, Apple Music, Beatport, Discogs)
- `isDirect: true` only for Bandcamp when a release-level URL is passed from MusicBrainz
- `getAffiliateConfig()` reads affiliate IDs from `$env/dynamic/private` — Cloudflare Pages runtime-only vars
- Amazon + Apple get affiliate-coded URLs when IDs present; Bandcamp/Beatport/Discogs are search fallbacks only

## Decisions Made
- `$env/dynamic/private` over `$env/static/private` — CF Pages env vars are runtime-only, not available at build time
- Discogs uses `/sell/list?q=...&type=release` — sends users to vinyl-for-sale listings, not the cataloging database (critical UX distinction)
- Apple campaign string `.slice(0, 40)` — Partnerize API enforces 40-char max
- Bandcamp `isDirect: true` only for release-level MusicBrainz URLs — artist-level Bandcamp URLs would send users to the artist homepage rather than a specific album
- Module is server-side only: `config.ts` imports from `$env/dynamic/private` which is unavailable in browser/Tauri contexts
