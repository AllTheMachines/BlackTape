    **src-tauri/src/ai/embeddings.rs:** sqlite-vec integration:
    - Register sqlite-vec extension with rusqlite BEFORE opening taste.db:
      ```rust
      use rusqlite::ffi::sqlite3_auto_extension;
      use sqlite_vec::sqlite3_vec_init;

      pub fn register_vec_extension() {
          unsafe {
              sqlite3_auto_extension(Some(std::mem::transmute(
                  sqlite3_vec_init as *const ()
              )));
          }
      }
      ```
    - Call `register_vec_extension()` in the taste_db init function BEFORE opening the connection.
    - After creating base tables, create the vec0 virtual table:
      ```sql
      CREATE VIRTUAL TABLE IF NOT EXISTS artist_embeddings USING vec0(
          embedding float[768]
      );
      ```
      Note: vec0 virtual tables use rowid as primary key. We need a mapping table:
      ```sql
      CREATE TABLE IF NOT EXISTS artist_embedding_map (
          rowid INTEGER PRIMARY KEY,
          artist_mbid TEXT NOT NULL UNIQUE,
          artist_name TEXT NOT NULL,
          embedded_at INTEGER NOT NULL
      );
      ```
      The rowid in artist_embedding_map matches the rowid in artist_embeddings.

    - Tauri commands for embedding operations:
      - `#[tauri::command] store_embedding(state, artist_mbid, artist_name, embedding: Vec<f32>)`:
        Check if artist already has embedding (via artist_embedding_map). If yes, update. If no, insert into both artist_embeddings and artist_embedding_map. Use a transaction.
      - `#[tauri::command] find_similar_artists(state, embedding: Vec<f32>, limit: usize)`:
        Query: `SELECT aem.artist_mbid, aem.artist_name, ae.distance FROM artist_embeddings ae JOIN artist_embedding_map aem ON aem.rowid = ae.rowid WHERE embedding MATCH ? ORDER BY distance LIMIT ?`. Serialize embedding as blob for the MATCH parameter. Return Vec of (mbid, name, distance).
      - `#[tauri::command] get_embedding(state, artist_mbid)`:
        Look up rowid from artist_embedding_map, then select from artist_embeddings. Return Option<Vec<f32>>.
      - `#[tauri::command] has_embedding(state, artist_mbid)`:
        Simple existence check in artist_embedding_map. Return bool.

    - For passing float vectors to sqlite-vec, use the zerocopy pattern from research:
      ```rust
      use zerocopy::IntoBytes;
      // Convert Vec<f32> to &[u8] for sqlite-vec blob parameter
      let embedding_bytes = embedding.as_bytes();
      ```
      Add `zerocopy = { version = "0.8", features = ["derive"] }` to Cargo.toml if not already added in Plan 01.

    - Update `src-tauri/src/ai/taste_db.rs`: Add favorite artist CRUD commands:
      - `#[tauri::command] add_favorite_artist(state, artist_mbid, artist_name, artist_slug)`: Insert into favorite_artists with current timestamp. Upsert (INSERT OR REPLACE).
      - `#[tauri::command] remove_favorite_artist(state, artist_mbid)`: DELETE from favorite_artists.
      - `#[tauri::command] get_favorite_artists(state)`: SELECT all, return Vec of favorites.
      - `#[tauri::command] is_favorite_artist(state, artist_mbid)`: Return bool.
    - Also add taste tag commands:
      - `#[tauri::command] get_taste_tags(state)`: SELECT all from taste_tags ORDER BY weight DESC.
      - `#[tauri::command] set_taste_tag(state, tag, weight, source)`: Upsert into taste_tags.
      - `#[tauri::command] remove_taste_tag(state, tag)`: DELETE from taste_tags.

  <action>
    **src/lib/taste/profile.svelte.ts:** Reactive taste profile state:
    ```typescript
    export interface TasteTag { tag: string; weight: number; source: string; }
    export interface TasteAnchor { artist_mbid: string; artist_name: string; }
    export interface FavoriteArtist { artist_mbid: string; artist_name: string; artist_slug: string; saved_at: number; }

    export const MINIMUM_TASTE_THRESHOLD = 5; // minimum favorite artists OR 20+ library tracks before recommendations

    export const tasteProfile = $state({
        tags: [] as TasteTag[],
        anchors: [] as TasteAnchor[],
        favorites: [] as FavoriteArtist[],
        isLoaded: false,
        hasEnoughData: false  // computed from favorites count + library tracks
    });
    ```
    - `loadTasteProfile()`: Loads tags, anchors, and favorites from Tauri invoke commands. Sets isLoaded = true. Computes hasEnoughData.
    - `computeHasEnoughData()`: Check tasteProfile.favorites.length >= MINIMUM_TASTE_THRESHOLD (5 favorites). Import libraryState and check tracks.length >= 20 as alternative. Either condition sufficient. Uses dynamic import for library to avoid circular deps.

    **src/lib/taste/favorites.ts:** Favorite artist operations (wraps Tauri invokes):
    - Uses dynamic imports for Tauri isolation (same pattern as library/scanner.ts).
    - `addFavorite(mbid, name, slug)`: invoke add_favorite_artist, update tasteProfile.favorites, trigger taste recomputation.
    - `removeFavorite(mbid)`: invoke remove_favorite_artist, update tasteProfile.favorites, trigger taste recomputation.
    - `isFavorite(mbid)`: Check tasteProfile.favorites array (local, no invoke needed after initial load).
    - `loadFavorites()`: invoke get_favorite_artists, populate tasteProfile.favorites.

    **src/lib/taste/signals.ts:** Taste signal computation:
    - `computeTasteFromLibrary()`: Read libraryState.tracks (dynamic import from library/store.svelte.ts). Extract unique artist tags by:
      1. Get all unique artists from library tracks (by artist name).
      2. For each unique artist, look up their tags from mercury.db via TauriProvider search.
      3. Build tag frequency map: each occurrence adds weight.
      4. Normalize weights to 0.0-1.0 range.
      5. Store results via set_taste_tag invoke with source='library'.
    - `computeTasteFromFavorites()`: Read tasteProfile.favorites. For each favorite:
      1. Look up artist tags from mercury.db.
      2. Add weight for each tag (favorites weighted 2x vs library).
      3. Store via set_taste_tag invoke with source='favorite'.
    - `recomputeTaste()`: Clears all taste_tags with source='library' or 'favorite' (preserve 'manual'). Runs both computations. Updates tasteProfile.tags. This is called when library changes or favorites change.
    - NOTE: This is best-effort. Not every library artist will match the mercury.db index. That's fine — partial data still builds a useful taste profile.

    **src/lib/taste/embeddings.ts:** Frontend embedding wrappers:
    - `generateEmbedding(text: string)`: Gets active AI provider from getAiProvider(). Calls provider.embed([text]). Returns the first embedding vector.
    - `embedArtist(mbid: string, name: string, tags: string, country: string)`: Builds text representation: `"${name}. genres: ${tags}. from ${country}"`. Calls generateEmbedding(). Stores result via invoke store_embedding.
    - `findSimilar(embedding: number[], limit: number)`: invoke find_similar_artists.
    - `getOrComputeEmbedding(mbid: string, name: string, tags: string, country: string)`: Check if embedding exists (invoke has_embedding). If yes, get it. If no, generate and store it. Returns embedding vector.
    - All functions gracefully handle AI not being ready (return null/empty).

    **src/lib/taste/index.ts:** Barrel export for all taste modules.
