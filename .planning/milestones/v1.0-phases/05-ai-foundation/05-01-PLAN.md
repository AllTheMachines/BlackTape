    **src/lib/ai/engine.ts:** AI provider interface and state management:
    ```typescript
    export interface CompletionOptions {
        maxTokens?: number;
        temperature?: number;
        systemPrompt?: string;
    }

    export interface AiProvider {
        complete(prompt: string, options?: CompletionOptions): Promise<string>;
        embed(texts: string[]): Promise<number[][]>;
        isReady(): Promise<boolean>;
    }

    // Module-level active provider state
    let activeProvider: AiProvider | null = null;

    export function setAiProvider(provider: AiProvider | null): void {
        activeProvider = provider;
    }

    export function getAiProvider(): AiProvider | null {
        return activeProvider;
    }
    ```

    **src/lib/ai/local-provider.ts:** LocalAiProvider implementation:
    - `GENERATION_PORT = 8847`, `EMBEDDING_PORT = 8848`
    - `complete()`: POST to `http://127.0.0.1:${GENERATION_PORT}/v1/chat/completions` with messages array (system + user). Parse response.choices[0].message.content. Handle fetch errors gracefully — return empty string on failure.
    - `embed()`: POST to `http://127.0.0.1:${EMBEDDING_PORT}/v1/embeddings` with input array. Parse response.data[].embedding. Handle fetch errors — return empty arrays on failure.
    - `isReady()`: GET `http://127.0.0.1:${GENERATION_PORT}/health`. Return `response.ok`. Catch errors → return false. NOTE: Only checks generation server. Embedding readiness can be checked separately if needed.

    **src/lib/ai/remote-provider.ts:** RemoteAiProvider implementation:
    - Constructor takes `apiKey: string`, `baseUrl: string`, `model: string`.
    - `complete()`: POST to `${baseUrl}/v1/chat/completions` with Authorization: Bearer header and model field. Same response parsing as local.
    - `embed()`: POST to `${baseUrl}/v1/embeddings` with same auth.
    - `isReady()`: Try a minimal completion request or just return true if apiKey is non-empty. Keep simple — remote API assumed available if configured.
    - NOTE: This uses the OpenAI-compatible API format. Works with OpenAI, Anthropic (via proxy), and other OpenAI-compatible providers.
