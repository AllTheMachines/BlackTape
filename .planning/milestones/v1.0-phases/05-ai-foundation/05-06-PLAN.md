  <action>
    **src/lib/components/TasteEditor.svelte:** Two-section editing panel:

    **Section 1: Tag Weights**
    - Title: "Your Taste Tags"
    - Subtitle: "These tags shape your recommendations. Adjust weights or add your own."
    - List of tags sorted by absolute weight (highest first):
      - Each tag row: tag name (left), source badge (small, muted — "library"/"favorite"/"manual"), weight slider or +/- buttons (right).
      - Weight display: visual bar or simple number (-1.0 to 1.0). Negative weights = suppressed genres. Use a horizontal slider (range input) styled to match theme. Slider thumb: var(--text-accent).
      - Source badge styling: library = muted pill, favorite = slightly brighter pill, manual = accent pill. Helps user understand where each tag came from.
      - Remove button (X) on each tag — removes from taste profile.
    - "Add tag" input at bottom: Text input + "Add" button. Adding a tag sets source='manual', weight=0.5.
    - When user adjusts a tag weight: call `set_taste_tag` invoke with source='manual' if it was user-adjusted (overrides original source). Update tasteProfile.tags reactively.
    - When user removes a tag: call `remove_taste_tag` invoke. Remove from tasteProfile.tags.

    **Section 2: Artist Anchors**
    - Title: "Artist Anchors"
    - Subtitle: "Pin artists that define your taste. These heavily influence recommendations."
    - List of pinned anchors:
      - Each row: artist name (left), remove button (right, X icon).
      - Artist name is plain text (not a link — this is settings, not browsing).
    - "Add anchor" input: Text input for artist name. On submit, search mercury.db for the artist (via TauriProvider). If found, add to taste_anchors. If not found, show subtle "Artist not found" message. Use dynamic import for DB search.
    - When anchor added: invoke `add_taste_anchor`, update tasteProfile.anchors.
    - When anchor removed: invoke `remove_taste_anchor`, update tasteProfile.anchors.

    **Reactivity:**
    - On mount: ensure tasteProfile is loaded (call loadTasteProfile if not loaded).
    - Changes to tags/anchors trigger `recomputeTaste()` from signals.ts — but ONLY the library/favorite sources are recomputed. Manual tags are preserved.

    **Update src/lib/taste/signals.ts:**
    - Modify `recomputeTaste()`: When clearing tags before recomputation, only clear source='library' and source='favorite'. Tags with source='manual' are NEVER touched by recomputation.
    - After recomputation, merge: library + favorite tags are recalculated, manual tags overlay on top.

    **Update src/lib/taste/profile.svelte.ts:**
    - Add `loadTasteProfile()` implementation if not already complete: loads tags, anchors, favorites from Tauri invokes.

    **Styling:**
    - Match Mercury dark theme. Sections have the same spacing as other settings sections.
    - Slider: thin track (var(--border-default) color), thumb (var(--text-accent)).
    - Input fields: same style as search bar / settings inputs (bg-elevated, border-default, border-radius).
    - Tag list: compact but readable. Each row has enough padding for comfortable interaction.
    - Source badges: tiny (0.65rem), rounded-full, background varies by source type.
    - Remove (X) buttons: small, muted, brighten on hover.
    - Empty states: "No tags yet — browse some artists and save your favorites" for tags, "No anchors — search for an artist to pin" for anchors.
