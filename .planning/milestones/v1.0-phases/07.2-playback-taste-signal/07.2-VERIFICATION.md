### Gaps Summary

One gap blocks full goal achievement:

**Export history to file is broken.** The `exportPlayHistory()` function in `history.ts` calls `invoke('export_play_history_to_path', {path, json})` after the user selects a save path via the Tauri dialog. This Rust command does not exist — it was described in SUMMARY-03 as "the real code path" but was never implemented in `taste_db.rs` and never registered in `lib.rs`. When the user picks a save file location, the invoke throws a Tauri error, which is caught by the inner try/catch. The outer try/catch's catch block (browser blob download) does NOT execute because the error is in the inner try/catch, not the outer one. Result: the dialog closes, no file is written, and the user gets no feedback.

The `export_play_history` Rust command (which returns JSON as a string) IS correctly implemented. Only the file-write step is missing.

**Fix required:** Implement `export_play_history_to_path(path: String, json: String) -> Result<(), String>` using `std::fs::write(path, json)` in `taste_db.rs`, and register it in `lib.rs`.

**Secondary note (warning, not blocker):** The first `$effect` in `EmbedPlayer.svelte` watching `loadedEmbeds['sc-*']` is dead code — no code path sets a SoundCloud key in `loadedEmbeds` (the SoundCloud template block renders via `soundcloudEmbedHtml` directly, not via click-to-load). The SoundCloud widget hook still works correctly via the second `$effect` that watches `soundcloudEmbedHtml`. This is a minor code cleanliness issue that does not affect behavior.
