## Decisions Made
- **SC Widget type split:** Plan's type definition used `Widget` twice with conflicting types — TypeScript duplicate identifier error. Fixed by declaring `SCWidget` (instance type) and `SCWidgetConstructor` (callable + `Events` property) separately.
- **Two $effect hooks for SC:** The SoundCloud embed has two render paths — `soundcloudEmbedHtml` renders immediately (server-fetched oEmbed), while click-to-load would set `loadedEmbeds['sc-*']`. Both paths need hooking. The 500ms setTimeout in the second effect lets `{@html ...}` inject the iframe into the DOM before querySelector runs.
- **Listening History not gated on AI:** Private mode and play count are meaningful regardless of whether AI is enabled. A user can use the local player without AI features and still want to see/manage their history.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed duplicate Widget identifier in SC type definition**
- **Found during:** Task 1 (npm run check)
- **Issue:** The plan's type cast had `Widget` declared twice in the same object type literal — once as a constructor function returning `{ bind }`, once as a `{ Events }` intersection. TypeScript rejected it: "Duplicate identifier 'Widget'."
- **Fix:** Split into `SCWidget` type alias (instance shape) and `SCWidgetConstructor` (callable + Events), then typed `sc.Widget` as `SCWidgetConstructor`
- **Files modified:** `src/lib/components/EmbedPlayer.svelte`
- **Commit:** `bc75b61`

**2. [Rule 1 - Bug] Removed @tauri-apps/plugin-fs dynamic import from history.ts**
- **Found during:** Task 2 (npm run build)
- **Issue:** Plan 02 used `@ts-ignore` to suppress the TypeScript error for the uninstalled `@tauri-apps/plugin-fs` package. This passed `npm run check` but failed `npm run build` — Rollup traverses dynamic imports and rejects unresolvable packages. The error only surfaced in Plan 03 because `ListeningHistory.svelte` imports `history.ts`, pulling it into the web bundle's traversal graph.
- **Fix:** Removed the `plugin-fs` try block entirely. The Rust `export_play_history_to_path` invoke was already the real code path (plugin-fs was never installed); the inner try/catch was dead code.
- **Files modified:** `src/lib/taste/history.ts`
- **Commit:** `4f77f24`
