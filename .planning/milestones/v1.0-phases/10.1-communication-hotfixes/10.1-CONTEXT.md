<decisions>
## Implementation Decisions

### DM Conversation List UI
- Each conversation row shows: pubkey + last message preview snippet
- Pubkey display format: Claude decides (truncated npub or similar — user left to Claude)
- Empty state: Simple text prompt — "No conversations yet. Start one below."
- Active/selected conversation is visually highlighted (distinct background or border on active row)

### New DM Initiation Flow
- Input field is fixed at the **bottom** of the DMs tab — always visible below the conversation list
- Submitting (Enter or button) navigates directly to a new empty DM thread with that pubkey
- Input accepts **both npub and hex pubkey** — detect format and normalize to hex internally
- Invalid pubkey shows an **inline error message below the input field** ("Invalid pubkey format")

### Taste Profile Publishing
- Trigger: Claude decides (auto-publish on Profile page load if Nostr is connected is a reasonable default)
- User feedback: **Subtle status indicator** near the Nostr section of the Profile page — non-intrusive "Published" / "Publishing..." label
- If not connected to a relay when publish is attempted: **silently skip** — no error shown to user, try again next time
- Status indicator placed **near the Nostr key display section** of Profile

### Export Command
- File format: Claude decides (JSON is the practical choice for play history interoperability)
- If target path already exists: Claude decides (overwrite is likely the pragmatic default)
- Return value from Rust command: Claude decides (path + count is most useful for UI)
- Frontend: **Yes — export button on Profile page near the play history section** — triggers the Rust command and shows the saved path

### Claude's Discretion
- Exact pubkey display truncation format (truncated npub vs short hex)
- Taste publish trigger timing (page load vs another automatic trigger)
- Export file format (JSON recommended)
- Export overwrite behavior
- Rust command return shape ({ path, count } recommended)
- Loading skeleton and exact spacing throughout

</decisions>
