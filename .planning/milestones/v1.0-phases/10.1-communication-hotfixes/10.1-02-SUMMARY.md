## Decisions Made
- No state parameter on `export_play_history_to_path` — the function only writes a file and does not access taste.db. Adding TasteDbState would risk a double-lock panic on the Mutex. Matches the existing `write_json_to_path` and `save_base64_to_file` pattern.
- Dynamic import for tasteProfile inside publishTasteProfile() body — avoids circular dependency between comms/ and taste/ modules at module load time.
- Session-level `tastePublished` module-level flag — kind:30078 is replaceable so re-publishing is harmless, but once per session is respectful to relays and sufficient for the ai-taste-bridge consumer.
