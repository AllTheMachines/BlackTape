#### 1. CSS Alias Visual Rendering (GAP-05)

#### 2. DM Conversation List End-to-End Flow

**Test:** In desktop app, open Chat (DMs tab). Verify the conversation list appears (or empty state "No conversations yet. Start one below."). Enter a valid hex pubkey or npub in the bottom input and press Enter or click "Start DM". Verify navigation to DM thread view. Click "← Conversations" to return.
**Expected:** Smooth navigation between conversation list and thread view; invalid pubkey shows inline "Invalid pubkey format" error.
**Why human:** Requires Nostr relay connection and actual keypair to fully exercise; navigation state is reactive and not statically traceable.

#### 3. Taste Profile Publishing Feedback

**Test:** In desktop app with a Nostr key configured, navigate to Profile page. Observe the Nostr section near identity block. Check for "Publishing…" then "Published" status indicator.
**Expected:** Brief "Publishing…" then "Published" (accent-colored) indicator appears; no error shown if not connected.
**Why human:** Requires live Nostr connection and published taste data; real-time state transitions cannot be verified statically.

#### 4. Export Play History Button

**Test:** In desktop app with play history, click "Export Play History" on Profile page. Confirm system save dialog appears, file is written, and status briefly shows "Exported."
**Expected:** File saved to chosen path; "Exported." message appears for ~3 seconds.
**Why human:** Requires Tauri file dialog (OS-level) and actual file system write; cannot be headlessly tested.
