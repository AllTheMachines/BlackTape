```
npm install @nostr-dev-kit/ndk@3.0.0 @nostr-dev-kit/ndk-svelte@2.4.48 nostr-tools@2.23.1 idb
```

Create `src/lib/comms/keypair.ts`:
- Import `openDB` from `idb`, `generateSecretKey` + `getPublicKey` from `nostr-tools`, `bytesToHex` from `@noble/hashes/utils`
- DB_NAME = `'mercury-comms'`, STORE = `'keypair'`
- Export `loadOrCreateKeypair(): Promise<{ privateKey: string; publicKey: string }>` — opens IndexedDB with upgrade creating the objectStore, reads `privateKey` from store, generates new key with `generateSecretKey()` if missing and stores the raw Uint8Array, derives `publicKey = getPublicKey(privKeyBytes)`, returns both as hex strings
- NOTE: Nostr uses secp256k1 (not in WebCrypto's supported curves), so store raw Uint8Array in IndexedDB — not as a non-extractable CryptoKey

Create `src/lib/comms/nostr.svelte.ts` (uses .svelte.ts for $state runes):
- Mercury relay list constant: `MERCURY_RELAYS = ['wss://nos.lol', 'wss://relay.damus.io', 'wss://nostr.mom', 'wss://relay.nostr.band']`
- Export `ndkState = $state({ ndk: null as NDK | null, pubkey: null as string | null, connected: false })`
- Export `initNostr(): Promise<void>` — calls `loadOrCreateKeypair()`, creates NDK instance with `explicitRelayUrls: MERCURY_RELAYS` and `enableOutboxModel: true`, creates `NDKPrivateKeySigner(privateKey)` and assigns to `ndk.signer`, calls `await ndk.connect()`, sets `ndkState.ndk`, `ndkState.pubkey`, `ndkState.connected = true`
- Use dynamic import for NDKPrivateKeySigner: `const { NDKPrivateKeySigner } = await import('@nostr-dev-kit/ndk')`

Create `src/lib/comms/index.ts` with re-exports:
```typescript
export { ndkState, initNostr } from './nostr.svelte.js';
export { loadOrCreateKeypair } from './keypair.js';
```

Anti-pattern to avoid: Do NOT store the private key in localStorage — IndexedDB only. Do NOT connect to more than 5 relays.

Create `src/lib/comms/unfurl.ts`:
- Export interface `UnfurlCard { title: string; description?: string; image?: string; url: string; }`
- Export `MERCURY_URL_PATTERN = /https?:\/\/[^\s]+\/(artist|release|kb)\/[^\s]+/g`
- Export `extractMercuryUrls(content: string): string[]` — uses matchAll with MERCURY_URL_PATTERN, returns array of matched URL strings
- Export `fetchUnfurlData(url: string): Promise<UnfurlCard | null>` — POSTs to `/api/unfurl` with JSON body `{ url }` and `Content-Type: application/json`, returns parsed JSON on 200, null on error (try/catch)
- Debounce NOTE: `fetchUnfurlData` is intentionally not debounced here — callers (MessageInput.svelte) apply their own 800ms debounce and URL validation before calling. Add code comment to this effect.

Create `src/routes/api/unfurl/+server.ts`:
- Import `unfurl` from `unfurl.js`
- Import `PUBLIC_SITE_URL` from `$env/static/public` if available, otherwise use `'https://mercury.app'` as fallback
- Export `POST` handler: parse `{ url }` from `request.json()`; if URL doesn't start with PUBLIC_SITE_URL, return 400 with `{ error: 'Not a Mercury URL' }`; call `await unfurl(url)`, return JSON with `{ title, description, image: meta.open_graph?.images?.[0]?.url, url }`; wrap in try/catch — on error return `{ url }` with 200 (graceful degradation, link still shows without preview)
- Set `Cache-Control: max-age=3600` response header (unfurl results don't change often)

Note: `unfurl.js` is a server-only package — it only runs in +server.ts routes, never imported in client-side .ts or .svelte files.
