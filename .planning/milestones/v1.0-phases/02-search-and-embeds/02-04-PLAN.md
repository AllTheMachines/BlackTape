  <name>Task 1: Create MusicBrainz API proxy, embed utilities, and bio fetcher</name>
  <files>src/routes/api/artist/[mbid]/links/+server.ts, src/lib/embeds/types.ts, src/lib/embeds/spotify.ts, src/lib/embeds/soundcloud.ts, src/lib/embeds/youtube.ts, src/lib/embeds/bandcamp.ts, src/lib/bio.ts</files>
  <action>
    1. Create `src/lib/embeds/types.ts`:
       - Export `PlatformType = 'bandcamp' | 'spotify' | 'soundcloud' | 'youtube'`
       - Export interface `PlatformLinks`:
         ```
         { bandcamp: string[], spotify: string[], soundcloud: string[], youtube: string[], wikipedia: string[], other: string[] }
         ```
       - Export interface `EmbedData`:
         ```
         { platform: PlatformType, url: string, embedUrl: string | null, embedHtml?: string }
         ```
       - Export `PLATFORM_PRIORITY: PlatformType[] = ['bandcamp', 'spotify', 'soundcloud', 'youtube']` (Bandcamp first per locked decision)

    2. Create `src/lib/embeds/spotify.ts`:
       - Export `spotifyEmbedUrl(url: string): string | null`
       - Parse Spotify URL: match `open.spotify.com/(artist|album|track|playlist)/([a-zA-Z0-9]+)`
       - Return `https://open.spotify.com/embed/{type}/{id}?theme=0` (theme=0 = dark mode)
       - Return null if URL doesn't match pattern

    3. Create `src/lib/embeds/youtube.ts`:
       - Export `youtubeEmbedUrl(url: string): string | null`
       - Match video URLs: `youtube.com/watch?v=`, `youtu.be/`, `youtube.com/embed/`
       - Return `https://www.youtube-nocookie.com/embed/{videoId}` for videos (privacy-friendly domain)
       - Return null for channel URLs (channels cannot be embedded — per research)
       - Export `isYoutubeChannel(url: string): boolean` — detect channel URLs for external link fallback

    4. Create `src/lib/embeds/soundcloud.ts`:
       - Export `soundcloudOembedUrl(url: string): string` — constructs the oEmbed API URL
         `https://soundcloud.com/oembed?url=${encodeURIComponent(url)}&format=json&maxheight=166`
       - The actual oEmbed fetch happens server-side in the artist page load function (not in this utility)
       - Export `isSoundcloudArtist(url: string): boolean` — detect artist/profile URLs vs track URLs

    5. Create `src/lib/embeds/bandcamp.ts`:
       - Export `bandcampExternalUrl(url: string): string` — returns the original URL (Bandcamp = external link only)
       - Export `isBandcampUrl(url: string): boolean` — detect Bandcamp URLs
       - NO embed URL generation. Bandcamp has NO oEmbed and requires album IDs not available from MusicBrainz URLs. Per locked decision: show "Listen on Bandcamp" external link fallback.

    6. Create `src/lib/bio.ts`:
       - Export `fetchWikipediaBio(wikiUrl: string): Promise<string | null>`
       - Extract page title from Wikipedia URL: match `wikipedia.org/wiki/(.+)`
       - Detect language from URL: match `//(\w+).wikipedia`
       - Fetch from `https://{lang}.wikipedia.org/api/rest_v1/page/summary/{title}`
       - Set `User-Agent: Mercury/0.1.0`
       - Return `data.extract` (plain text first paragraph) or null on failure
       - Wrap in try/catch — return null on any error. Bio is best-effort, not critical.

    7. Create `src/routes/api/artist/[mbid]/links/+server.ts`:
       - Export `GET` request handler
       - Read `mbid` from params
       - Validate mbid format (UUID pattern)
       - Check Cloudflare Cache API first:
         - Cache key: `https://mercury.internal/api/artist/${mbid}/links`
         - If cached, return cached response
       - Rate limit: Track last MusicBrainz request timestamp. Wait if less than 1100ms since last request.
       - Fetch from MusicBrainz: `https://musicbrainz.org/ws/2/artist/${mbid}?inc=url-rels&fmt=json`
       - Set headers: `User-Agent: Mercury/0.1.0 (https://github.com/user/mercury)`, `Accept: application/json`
       - Extract platform links from `data.relations` array:
         - For each relation where `target-type === 'url'`, get `rel.url.resource`
         - Categorize by domain: bandcamp.com → bandcamp, open.spotify.com → spotify, soundcloud.com → soundcloud, youtube.com/youtu.be → youtube, wikipedia.org → wikipedia, everything else → other
       - Return JSON with `PlatformLinks` shape
       - Set response headers: `Cache-Control: public, max-age=86400` (24hr)
       - Store in Cloudflare cache via `cache.put()`
       - Handle MusicBrainz errors: return 502 on API failure, 404 on artist not found
  </action>

Verify embed URL transformers with manual spot checks: `spotifyEmbedUrl('https://open.spotify.com/artist/4Z8W4fKeB5YxbusRsdQVPb')` returns embed URL. `youtubeEmbedUrl('https://www.youtube.com/watch?v=dQw4w9WgXcQ')` returns nocookie embed URL. `bandcampExternalUrl()` returns original URL.</verify>
  <done>MusicBrainz API proxy fetches and caches external links with rate limiting. Embed utilities transform platform URLs into embed-ready URLs. Bio fetcher retrieves Wikipedia summaries. All utilities handle errors gracefully.</done>

    1. Create `src/routes/artist/[slug]/+page.server.ts`:
       - Export `load` function (PageServerLoad)
       - Get `slug` from params, DB from `platform.env.DB`
       - Call `getArtistBySlug(db, slug)` — if null, `throw error(404, 'Artist not found')`
       - Fetch external links: `fetch(\`/api/artist/${artist.mbid}/links\`)` (internal fetch)
       - Parse links response as `PlatformLinks`
       - For SoundCloud URLs: attempt oEmbed fetch server-side to get embed HTML
         - Fetch `soundcloudOembedUrl(url)`, parse JSON, extract `html` field
         - If fails, SoundCloud becomes external link fallback
       - For Wikipedia URLs: call `fetchWikipediaBio()` with the first Wikipedia URL
       - Return: `{ artist, links, bio, soundcloudEmbedHtml }`
       - Handle errors: if MusicBrainz proxy fails, return artist with empty links (page still renders with DB data)
       - Bio and links are best-effort — page works without them

    2. Create `src/lib/components/ExternalLink.svelte`:
       - Props: `url: string`, `platform: PlatformType | 'other'`, `label?: string`
       - Renders as `<a href={url} target="_blank" rel="noopener noreferrer">`
       - Shows platform name and icon/color: "Listen on Bandcamp" (--bandcamp-color), "Listen on Spotify" (--spotify-color), etc.
       - For non-platform URLs: generic "Visit" label
       - Style: pill-shaped button, platform color as accent, --bg-elevated background
       - Hover state: background lightens

    3. Create `src/lib/components/EmbedPlayer.svelte`:
       - Props: `links: PlatformLinks`, `soundcloudEmbedHtml?: string`
       - Render embeds in priority order: Bandcamp > Spotify > SoundCloud > YouTube (per PLATFORM_PRIORITY)
       - For each platform with URLs:
         - **Bandcamp**: Always render as ExternalLink (no embed available per research)
         - **Spotify**: If URL parseable by spotifyEmbedUrl → render iframe (dark theme, height ~352px for artist, ~80px for track). If not parseable → ExternalLink fallback.
         - **SoundCloud**: If soundcloudEmbedHtml provided → render with `{@html}`. Otherwise → ExternalLink fallback.
         - **YouTube**: If URL parseable by youtubeEmbedUrl (video URL) → render iframe (16:9 aspect ratio, responsive). If channel URL → ExternalLink "Visit YouTube Channel". Consider using lite-youtube-embed for performance (import if installed, standard iframe as fallback).
       - Lazy loading: All iframes get `loading="lazy"` attribute. Additionally, use click-to-load pattern — show a styled placeholder button ("Play on Spotify") that reveals the iframe on click. This avoids loading heavy iframes until the user explicitly wants to listen.
       - No-embed fallback: If a platform has URLs but none are embeddable, show ExternalLink instead. No empty states. Every platform with a URL shows *something*.
       - Iframe styling: full width of container, themed border (--border-subtle), --card-radius border-radius

    4. Create `src/routes/artist/[slug]/+page.svelte`:
       - Import: TagChip, EmbedPlayer, ExternalLink, PROJECT_NAME from config
       - `<svelte:head>`: title = "{artist.name} — {PROJECT_NAME}", meta description with tags
       - **Balanced two-column layout** (info left, embeds right — per locked decision):
         - On desktop (>768px): two columns, roughly equal width (CSS grid: `1fr 1fr` or `5fr 4fr`)
         - On mobile: single column, info on top, embeds below
       - **Left column (Info):**
         - Artist name (h1, large, --text-accent)
         - Artist type + country (e.g., "Group — GB") in --text-secondary
         - Bio snippet: if available, show first paragraph of Wikipedia summary. Style as --text-primary, normal font size. If unavailable, omit section entirely (no "No bio available" message).
         - Tags: row of TagChip components (all tags, not just top 5), wrapped in a flex container with gap
         - External links: row of ExternalLink buttons for any URLs in the `other` category
       - **Right column (Embeds):**
         - EmbedPlayer component with the platform links
         - If no embeddable links exist at all: show row of ExternalLink buttons for all platforms with URLs
       - Style with theme variables. --bg-base background, generous padding, --max-width container
       - Information-dense: pack data in, but readable. No hero images, no decorative elements.
