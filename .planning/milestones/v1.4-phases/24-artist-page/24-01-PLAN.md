Add a new fetch block in the `load` function after the existing links fetch. Call:
`https://musicbrainz.org/ws/2/artist/{artist.mbid}?inc=artist-rels+label-rels&fmt=json`
with the same `USER_AGENT` header pattern already used in this file.

Define TypeScript types inline (no separate types file needed):
```ts
interface ArtistRelationships {
  members: Array<{ name: string; mbid: string }>;
  influencedBy: Array<{ name: string; mbid: string }>;
  influenced: Array<{ name: string; mbid: string }>;
  labels: string[];
}
```

Parse the `relations` array from the MB response:
- `target-type === 'artist'` AND `type === 'member of band'` AND `direction === 'backward'` → member of this band (push `{ name: rel.artist.name, mbid: rel.artist.id }`)
- `target-type === 'artist'` AND `type === 'influenced by'` AND `direction === 'forward'` → influencedBy (push artist)
- `target-type === 'artist'` AND `type === 'influenced by'` AND `direction === 'backward'` → influenced (push artist)
- `target-type === 'label'` → push `rel.label?.name` into labels array (as plain string)

Wrap the entire fetch in try/catch — on failure, assign empty relationships object. Add `relationships: ArtistRelationships` to the return value (default: `{ members: [], influencedBy: [], influenced: [], labels: [] }`).

**Structure — three conditional sections rendered only when they have data:**

**Members section** (if `relationships.members.length > 0`):
```svelte
<section class="rel-section">
  <h3>Members</h3>

    {#if relationships.members.length > 20 && !showAllMembers}
```
