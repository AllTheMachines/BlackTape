Add a new credits fetch to the load function. The MusicBrainz endpoint for release-level artist relationships:
`https://musicbrainz.org/ws/2/release?release-group={releaseGroupMbid}&inc=artist-rels&limit=1&fmt=json`

This returns releases in a group. Take the first release and parse its `relations` array:
- Filter to `target-type === 'artist'`
- Relevant relationship types to include: `'producer'`, `'engineer'`, `'mix'`, `'lyricist'`, `'composer'`, `'performer'`, `'instrument'`, `'vocal'`
- For each relevant relation, extract: `{ role: rel.type, name: rel.artist.name, mbid: rel.artist.id }`

Define the type:
```ts
type CreditEntry = { role: string; name: string; mbid: string; slug: string | null };
```

After collecting the credits array, perform slug lookups against the local DB:
```ts
const { getProvider } = await import('$lib/db/provider');
const provider = await getProvider();
const creditsWithSlugs = await Promise.all(
  rawCredits.map(async (c) => {
    try {
      const row = await provider.get<{ slug: string }>(
        'SELECT slug FROM artists WHERE mbid = ?',
        c.mbid
      );
      return { ...c, slug: row?.slug ?? null };
    } catch {
      return { ...c, slug: null };
    }
  })
);
```

Return `credits: creditsWithSlugs` in the load function. Default to `credits: []` if the fetch fails.

Note: The existing `release.credits` field in the page data comes from the existing credits section in the release response (from `rel.credits`). The new `credits` array at top-level is separate and more detailed. Both can coexist. The new one is what the new Credits UI section uses.
