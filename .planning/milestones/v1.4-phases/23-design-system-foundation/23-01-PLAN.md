  <action>
**Step A — Create `src/lib/components/Titlebar.svelte`:**

Build a custom titlebar that replaces the native OS window chrome. It must:
- Span the full window width, height 28px, `background: var(--bg-1)`, `border-bottom: 1px solid var(--b-1)`
- Left: logo mark — project name in uppercase amber (`color: var(--acc)`, 10px, 700 weight, 0.12em letter-spacing)
- Right: three window control buttons — minimize (−), maximize (□), close (×)
  - Each button: 28×28px, `background: transparent`, `border: none`, `color: var(--t-3)`, hover: `background: var(--bg-4)`, close hover: `background: #c0392b; color: white`
- The titlebar drag region: use `data-tauri-drag-region` attribute on the titlebar wrapper div so Tauri recognizes the drag area. The window control buttons must NOT be inside the drag region (they need to be clickable).
- Import `{ getCurrentWindow }` from `@tauri-apps/api/window` to get `appWindow`. Call `appWindow.minimize()`, `appWindow.toggleMaximize()`, `appWindow.close()` on respective button clicks.
- Only render window controls inside `{#if isTauri()}` guard.
- Import `isTauri` from `$lib/platform`.
- The component is always rendered (for layout), but window controls are Tauri-only.

**Step B — Update `src-tauri/tauri.conf.json`:**

In the `app.windows[0]` object, add `"decorations": false`. This removes the native OS titlebar.

**Step C — Mount Titlebar in `src/routes/+layout.svelte`:**

Import `Titlebar` from `$lib/components/Titlebar.svelte`. In the layout template (outside the `{#if isEmbed}` guard), render `<Titlebar />` as the very first element in the `{:else}` block (before the loading bar and `<header>`). Only render it when `tauriMode` is true:

```svelte
{#if tauriMode}
  <Titlebar />
{/if}
```

The layout should shift content down by 28px when the titlebar is visible — set `padding-top: 28px` on the root or handle via grid. The simplest approach: wrap the entire layout in a flex column and place Titlebar first.
