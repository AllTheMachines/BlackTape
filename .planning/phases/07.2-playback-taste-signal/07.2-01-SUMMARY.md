## Decisions Made
- **play_history in taste.db, not library.db:** Play history is a taste signal — belongs alongside taste_tags, favorite_artists, taste_anchors. library.db is a track catalog; taste.db is the user's relationship with those tracks.
- **70% threshold is frontend-enforced:** record_play is a generic write command. The caller (TypeScript) decides when a qualifying play has occurred. Threshold adjustable without Rust recompile.
- **private_listening default seeded now:** The default ("false") is needed in ai_settings before Plan 02 implements the gating UI. Seeding it now with INSERT OR IGNORE means existing users get the default without any migration.
