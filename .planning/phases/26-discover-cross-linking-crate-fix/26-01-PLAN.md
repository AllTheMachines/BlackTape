  <name>Task 1: Add uniqueness_score to ArtistResult + getDiscoveryArtists query with filter support</name>
  <files>src/lib/db/queries.ts</files>
  <action>
    1. Extend `ArtistResult` interface to add `uniqueness_score: number | null` field.

    2. Add a new exported query function `getDiscoveryArtists(db, filters, limit)` that accepts:
       ```ts
       interface DiscoverFilters {
         tags?: string[];
         country?: string;
         era?: string; // e.g. "60s", "70s", "80s", "90s", "00s", "10s", "20s"
       }
       ```
       The era maps to decade ranges:
       - '60s': begin_year BETWEEN 1960 AND 1969
       - '70s': begin_year BETWEEN 1970 AND 1979
       - '80s': begin_year BETWEEN 1980 AND 1989
       - '90s': begin_year BETWEEN 1990 AND 1999
       - '00s': begin_year BETWEEN 2000 AND 2009
       - '10s': begin_year BETWEEN 2010 AND 2019
       - '20s': begin_year BETWEEN 2020 AND 2029

       The query selects:
       ```sql
       SELECT
         a.id, a.mbid, a.name, a.slug, a.country,
         (SELECT GROUP_CONCAT(at2.tag, ', ') FROM artist_tags at2 WHERE at2.artist_id = a.id) AS tags,
         ROUND(
           COALESCE(
             (SELECT AVG(1.0 / NULLIF(ts.artist_count, 0)) * 1000
              FROM artist_tags at3
              JOIN tag_stats ts ON ts.tag = at3.tag
              WHERE at3.artist_id = a.id),
             0
           ),
           2
         ) AS uniqueness_score
       FROM artists a
       [JOIN artist_tags ON artist_tags.artist_id = a.id AND artist_tags.tag IN (...) -- if tags]
       WHERE a.id IN (SELECT DISTINCT artist_id FROM artist_tags)
       [AND a.country = ? -- if country]
       [AND a.begin_year BETWEEN ? AND ? -- if era]
       [GROUP BY a.id HAVING COUNT(DISTINCT ...) = tags.length -- if tags intersection]
       ORDER BY uniqueness_score DESC
       LIMIT ?
       ```

       When tags are provided, use the same JOIN intersection pattern as `getArtistsByTagIntersection` (join artist_tags for each tag, GROUP BY a.id, HAVING COUNT = tags.length).

       When no filters provided, fall back to ORDER BY discovery_score DESC (same as getDiscoveryRankedArtists — pick those artists first).

       Keep the existing `getDiscoveryRankedArtists` function intact (it may be used elsewhere).

    3. The `uniqueness_score` field on `ArtistResult` will be null for existing queries that don't compute it (backwards compatible — existing callers simply don't use the field).
  </action>
