**src/lib/library/types.ts** — TypeScript interfaces:
- `LocalTrack`: mirrors the Rust `LocalTrack` struct — `id: number`, `path: string`, `title: string | null`, `artist: string | null`, `album: string | null`, `album_artist: string | null`, `track_number: number | null`, `disc_number: number | null`, `genre: string | null`, `year: number | null`, `duration_secs: number`, `file_modified: number`, `created_at: string`
- `MusicFolder`: `id: number`, `path: string`, `added_at: string`
- `ScanProgress`: `scanned: number`, `total: number`, `current_file: string`
- `LibraryAlbum`: derived grouping — `name: string`, `artist: string`, `year: number | null`, `tracks: LocalTrack[]`
- Note: All field names use snake_case to match Rust's serde serialization (Tauri serializes Rust structs as snake_case by default)

**src/lib/library/scanner.ts** — Tauri invoke wrappers:
- Use dynamic imports to keep Tauri deps out of web build:
  ```typescript
  async function getInvoke() {
    const { invoke } = await import('@tauri-apps/api/core');
    return invoke;
  }
  ```
- Implement `scanMusicFolder(path: string, onProgress: (p: ScanProgress) => void): Promise<number>`:
  - Dynamically import `Channel` from `@tauri-apps/api/core`
  - Create a Channel, set `channel.onmessage = onProgress`
  - Call `invoke('scan_folder', { path, onProgress: channel })`
  - Return the total scanned count
- Implement `getLibraryTracks(): Promise<LocalTrack[]>`:
  - Call `invoke('get_library_tracks')`
- Implement `getMusicFolders(): Promise<MusicFolder[]>`:
  - Call `invoke('get_music_folders')`
- Implement `addMusicFolder(path: string): Promise<void>`:
  - Call `invoke('add_music_folder', { path })`
- Implement `removeMusicFolder(path: string): Promise<void>`:
  - Call `invoke('remove_music_folder', { path })`
- Implement `pickMusicFolder(): Promise<string | null>`:
  - Dynamic import `open` from `@tauri-apps/plugin-dialog`
  - Call `open({ directory: true, multiple: false, title: 'Select your music folder' })`
  - Return selected path or null

**src/lib/library/store.ts** — Reactive library state:
- Create reactive state with Svelte 5 runes:
  ```typescript
  export const libraryState = $state({
    tracks: [] as LocalTrack[],
    folders: [] as MusicFolder[],
    isScanning: false,
    scanProgress: null as ScanProgress | null,
    isLoaded: false,
    sortBy: 'artist' as 'artist' | 'album' | 'title' | 'added',
    sortAsc: true,
  });
  ```
- Implement `loadLibrary()`:
  - Call `getLibraryTracks()` and `getMusicFolders()` from scanner.ts
  - Update libraryState.tracks, libraryState.folders, libraryState.isLoaded = true
- Implement `scanFolder(path: string)`:
  - Set `libraryState.isScanning = true`
  - Call `addMusicFolder(path)` to register the folder
  - Call `scanMusicFolder(path, progress => libraryState.scanProgress = progress)`
  - After scan completes, reload tracks via `getLibraryTracks()`
  - Set `libraryState.isScanning = false`, `libraryState.scanProgress = null`
- Implement `groupByAlbum(tracks: LocalTrack[]): LibraryAlbum[]`:
  - Group tracks by `album_artist || artist` + `album`
  - Sort tracks within each album by disc_number then track_number
  - Return sorted array of LibraryAlbum objects
- Implement `getSortedTracks()`: returns tracks sorted by current sortBy/sortAsc settings
