  <name>Task 1: Extend taste.db schema with identity and collections tables</name>
  <files>src-tauri/src/ai/taste_db.rs</files>
  <action>
In `init_taste_db()`, extend the `execute_batch` call to add the following tables after the existing `play_history` index. Append to the existing SQL string — do NOT replace existing tables:

```sql
CREATE TABLE IF NOT EXISTS user_identity (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS collections (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);

CREATE TABLE IF NOT EXISTS collection_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    collection_id TEXT NOT NULL REFERENCES collections(id) ON DELETE CASCADE,
    item_type TEXT NOT NULL,
    item_mbid TEXT NOT NULL,
    item_name TEXT NOT NULL,
    item_slug TEXT,
    added_at INTEGER NOT NULL,
    UNIQUE(collection_id, item_type, item_mbid)
);
CREATE INDEX IF NOT EXISTS idx_collection_items_collection ON collection_items(collection_id);
CREATE INDEX IF NOT EXISTS idx_collection_items_mbid ON collection_items(item_mbid);
```

Then add the following Tauri commands to the same file (after the existing favorite_artists CRUD section):

**Identity commands:**
- `get_identity_value(key: String, state) -> Result<Option<String>, String>` — SELECT value FROM user_identity WHERE key = ?1
- `set_identity_value(key: String, value: String, state) -> Result<(), String>` — INSERT INTO user_identity (key, value) VALUES (?1, ?2) ON CONFLICT(key) DO UPDATE SET value = excluded.value
- `get_all_identity(state) -> Result<HashMap<String, String>, String>` — SELECT key, value FROM user_identity, collect into HashMap

**Collections commands:**
- `get_collections(state) -> Result<Vec<Collection>, String>` — SELECT id, name, created_at, updated_at FROM collections ORDER BY created_at ASC. Define `Collection` struct: `{ id: String, name: String, created_at: i64, updated_at: i64 }` with `#[derive(Debug, Serialize)]`
- `create_collection(name: String, state) -> Result<String, String>` — generate id as `format!("{}", now_millis())`, INSERT INTO collections. Return the new id.
- `delete_collection(id: String, state) -> Result<(), String>` — DELETE FROM collections WHERE id = ?1. ON DELETE CASCADE handles items.
- `rename_collection(id: String, name: String, state) -> Result<(), String>` — UPDATE collections SET name = ?2, updated_at = ?3 WHERE id = ?1

**Collection items commands:**
- `get_collection_items(collection_id: String, state) -> Result<Vec<CollectionItem>, String>` — SELECT * FROM collection_items WHERE collection_id = ?1 ORDER BY added_at DESC. Define `CollectionItem` struct: `{ id: i64, collection_id: String, item_type: String, item_mbid: String, item_name: String, item_slug: Option<String>, added_at: i64 }` with `#[derive(Debug, Serialize)]`
- `add_collection_item(collection_id: String, item_type: String, item_mbid: String, item_name: String, item_slug: Option<String>, state) -> Result<(), String>` — INSERT OR IGNORE INTO collection_items. Also UPDATE collections SET updated_at = now WHERE id = collection_id.
- `remove_collection_item(collection_id: String, item_type: String, item_mbid: String, state) -> Result<(), String>` — DELETE FROM collection_items WHERE collection_id = ?1 AND item_type = ?2 AND item_mbid = ?3
- `is_in_collection(item_type: String, item_mbid: String, state) -> Result<Vec<String>, String>` — SELECT collection_id FROM collection_items WHERE item_type = ?1 AND item_mbid = ?2. Returns list of collection IDs containing this item.
- `get_all_collection_items(state) -> Result<Vec<CollectionItem>, String>` — SELECT * FROM collection_items ORDER BY added_at DESC (for full JSON export)

**Batch artist matching helper command (for import flows):**
- `match_artists_batch(names: Vec<String>, state: tauri::State<'_, crate::MercuryDbState>) -> Result<Vec<MatchResult>, String>` — NOTE: This command takes MercuryDbState (mercury.db), NOT TasteDbState. Define `MatchResult` struct: `{ name: String, artist_mbid: Option<String>, artist_slug: Option<String> }`. For each name in names: SELECT mbid, slug FROM artists WHERE LOWER(name) = LOWER(?1) LIMIT 1. Collect results. If no match, return name with None fields. This is needed in `lib.rs` which has access to both states — add it there rather than in taste_db.rs if the signature requires both states simultaneously.

**Fingerprint export command:**
- `save_base64_to_file(path: String, data: String) -> Result<(), String>` — decode base64 using `base64::engine::general_purpose::STANDARD.decode(&data)`, then `std::fs::write(&path, bytes)`. Requires adding `base64 = "0.22"` to `src-tauri/Cargo.toml` `[dependencies]`.

**General-purpose JSON file write command (for full-data export in Plan 03):**
- `write_json_to_path(path: String, json: String) -> Result<(), String>` — simply `std::fs::write(&path, json.as_bytes())`. This is a clean, general-purpose command that accepts an arbitrary JSON string — do NOT reuse `export_play_history_to_path` for this, as that command's signature is `(path: String, state)` and does not accept a pre-serialised `json` string parameter.

Add `base64 = "0.22"` to `src-tauri/Cargo.toml` in the `[dependencies]` section.
