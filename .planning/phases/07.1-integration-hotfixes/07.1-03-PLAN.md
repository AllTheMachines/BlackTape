**Change 2 — Add discover link at bottom:**

After the closing `</div>` of the entire `.genre-page` div's last section (the mini genre graph section, or key artists / related genres if graph is absent), but still inside the `.genre-page` div, add:

```svelte
{#if data.genre.mb_tag}
    <div class="discover-footer">
        <a href="/discover?tags={data.genre.mb_tag}" class="discover-link">
            Explore {data.genre.name} in Discover
        </a>
    </div>
{/if}
```

Place this as the very last element inside the `<div class="genre-page">` container.

**CRITICAL pitfall (from RESEARCH.md Pitfall 4):** Wrap BOTH links in `{#if data.genre.mb_tag}`. The genres table has nullable `mb_tag`. A null/undefined `mb_tag` would produce `/discover?tags=null` — a broken URL that returns no results. Only render the link when the tag is a valid string.

```css
}
```

**Link label:** The label uses the actual genre/scene name dynamically: `"Explore {data.genre.name} in Discover"`. No generic label — per user decision.

**URL construction:** Template literal `/discover?tags=${data.genre.mb_tag}`. No URL builder needed — the TagFilter component on the Discover page reads `?tags=` from `page.url.searchParams` and pre-populates the filter automatically.
