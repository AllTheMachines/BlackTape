  <action>
The KB `/kb` page currently renders a static graph from server/page.ts data. After the GAP-01 layout fix lands, `loadTasteProfile()` fires on startup — but SvelteKit page loads are not reactive, so the Tauri path in `kb/+page.ts` reads `tasteProfile.tags` synchronously before the profile resolves, always getting `[]`. The page needs an in-component reactive refetch.

**Changes to `src/routes/kb/+page.svelte`:**

**1. Update the script block:**

Add imports for `tasteProfile`, `isTauri`, and `$state` override for graph:

```typescript
<script lang="ts">
    import GenreGraph from '$lib/components/GenreGraph.svelte';
    import type { PageData } from './$types';
    import { PROJECT_NAME } from '$lib/config';
    import { tasteProfile } from '$lib/taste/profile.svelte';
    import { isTauri } from '$lib/platform';

    let { data }: { data: PageData } = $props();

    // Local graph state — starts with server data, updates reactively when taste loads
    let graph = $state(data.graph);

    // Re-fetch personalized graph when taste profile becomes available (Tauri only)
    $effect(() => {
        if (tasteProfile.isLoaded && isTauri()) {
            loadPersonalizedGraph();
        }
    });

    async function loadPersonalizedGraph() {
        try {
            const { getProvider } = await import('$lib/db/provider');
            const { getStarterGenreGraph } = await import('$lib/db/queries');
            const db = await getProvider();
            const tasteTags = tasteProfile.tags.map((t) => t.tag).slice(0, 5);
            graph = await getStarterGenreGraph(db, tasteTags);
        } catch {
            // Keep current graph — best-effort
        }
    }
</script>
```

**2. Update the template — use local `graph` state instead of `data.graph`:**

Change `data.graph.nodes.length` to `graph.nodes.length` and pass `graph.nodes`/`graph.edges` to `GenreGraph`.

**3. Add skeleton loading indicator in the `kb-header` section:**

After the `<p class="kb-subtitle">` paragraph, add a taste-loading indicator (per user decision: skeleton loaders while loading):

```svelte
{#if isTauri() && !tasteProfile.isLoaded}
    <div class="taste-loading">
        <div class="skeleton-line" style="width: 45%;"></div>
    </div>
{/if}
```

**4. Add skeleton CSS:**

In the `<style>` block, add the canonical skeleton pattern (copy from existing AiRecommendations.svelte pattern):

```css

    margin: var(--space-xs, 0.5rem) 0 var(--space-md, 1rem);
}

.skeleton-line {
    height: 0.875rem;
    background: var(--bg-elevated);
    border-radius: 4px;
    animation: skeleton-pulse 1.5s ease-in-out infinite;
}

@keyframes skeleton-pulse {
    0%, 100% { opacity: 0.4; }
    50% { opacity: 0.8; }

```

**CRITICAL pitfall:** The `$effect` depends on `tasteProfile.isLoaded` as a reactive dependency. This is a Svelte 5 rune — it must be inside a `$effect()` block, not `onMount`. Using `onMount` would not be reactive and would miss the async load completion.

**CRITICAL pitfall:** `graph` must be declared with `$state()` (not `let`) so that when `loadPersonalizedGraph()` assigns `graph = await getStarterGenreGraph(...)`, Svelte reactivity picks up the change and re-renders the GenreGraph component.

**Changes to `src/routes/explore/+page.svelte`:**

**1. In the `{:else}` block (Tauri + AI ready — the full explore interface), add two new conditional blocks:**

The existing structure in the full explore block is:
```svelte
<div class="page-header">
    <h1>Explore</h1>
```

Replace with this expanded structure:
```svelte
<div class="page-header">
    <h1>Explore</h1>
    {#if !tasteProfile.isLoaded}
        <!-- Skeleton: profile is loading — show placeholder for hint area -->
        <div class="taste-hint-skeleton">
            <div class="skeleton-line" style="width: 55%;"></div>
        </div>
    {:else if topTasteTags.length > 0}
        <!-- Normal state: taste data available -->
        <p class="taste-hint">
            Your taste leans toward {topTasteTags.join(', ')}
        </p>
    {:else}
        <!-- Empty taste state: loaded but no data -->
        <p class="taste-empty">
            <a href="/">Search for artists</a> and save favorites to get personalized exploration.
        </p>
    {/if}
</div>
```

**CRITICAL pitfall (from RESEARCH.md Pitfall 6):** Use `tasteProfile.isLoaded` for the skeleton check — NOT `topTasteTags.length === 0`. Both loading AND empty profiles have `tags.length === 0`. Using length check would show skeleton to new users forever. The correct states are:
- `!tasteProfile.isLoaded` → skeleton (profile is still loading)
- `tasteProfile.isLoaded && tags.length > 0` → taste hint (normal state)
- `tasteProfile.isLoaded && tags.length === 0` → empty CTA ("Tag some artists...")

`tasteProfile` is already imported in the existing explore page script block (line 6: `import { tasteProfile } from '$lib/taste/profile.svelte'`). No new import needed.
