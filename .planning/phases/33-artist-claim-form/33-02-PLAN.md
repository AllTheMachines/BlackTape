<task type="auto">
  <name>Task 1: Create /claim form page</name>
  <files>src/routes/claim/+page.svelte</files>
  <action>
Create `src/routes/claim/+page.svelte`. No `+page.ts` loader — data arrives via URL params at runtime.

**Script block (Svelte 5 runes):**
- Import `page` from `$app/stores`, `onMount` from `svelte`, `PROJECT_NAME` from `$lib/config`
- Constant: `const WORKER_URL = 'https://blacktape-signups.theaterofdelays.workers.dev/claim';`
- State: `artistName = $state('')`, `artistReadOnly = $state(false)`, `fromSlug = $state('')`, `email = $state('')`, `message = $state('')`, `submitted = $state(false)`, `loading = $state(false)`, `error = $state('')`
- `onMount`: read `$page.url.searchParams.get('artist')` — if present, set artistName + set artistReadOnly=true; read `$page.url.searchParams.get('from')` into fromSlug
- `handleSubmit(e)`: preventDefault, clear error, validate (artistName non-empty, email regex `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`, message non-empty), set loading=true, POST JSON `{artistName, email, message}` to WORKER_URL, on ok set submitted=true, on error set error message, finally loading=false

**Template:**
When `submitted` is false, render:
- `<svelte:head><title>Claim your page — {PROJECT_NAME}</title></svelte:head>`
- Page wrapper `<div class="claim-page">`
- Heading: "Claim your artist page"
- Intro paragraph: "If this is your page, you can claim it. Once claimed, we'll be in touch to connect your profile, add links, and help your music reach the right people." (Claude's discretion on exact wording)
- Form with `onsubmit={handleSubmit}`:
  - Artist name input: `type="text"`, bound to `artistName`, `readonly={artistReadOnly}`, label "Artist name"
  - Email input: `type="email"`, bound to `email`, required, label "Your email"
  - Message textarea: bound to `message`, required, placeholder "Link your website, Bandcamp, social profiles, or any proof you're this artist", label "How should we verify this is you?"
  - Error block: `{#if error}<p class="claim-error">{error}</p>{/if}`
  - Submit button: `disabled={loading}`, text `{loading ? 'Sending...' : 'Submit claim'}`

When `submitted` is true, render confirmation replacing the form:
- `<div class="claim-confirmation">`
- Heading: "Thanks — we'll be in touch."
- Paragraph: "Your claim for {artistName} has been received. We'll review it and reach out to you at {email}."
- Back link: if `fromSlug` is non-empty, show `<a href="/artist/{fromSlug}">Back to {artistName}'s page</a>`; otherwise show `<a href="/search?q={encodeURIComponent(artistName)}">Search for {artistName}</a>`

**Styles:**
- `.claim-page`: `max-width: 480px; margin: 2rem auto; padding: 0 1rem;`
- Form labels: `display: block; margin-bottom: 0.25rem; font-size: 0.875rem; color: var(--t-2);`
- Inputs/textarea: `width: 100%; padding: 0.5rem 0.75rem; background: var(--bg-2); border: 1px solid var(--b-1); border-radius: 4px; color: var(--t-1); font-size: 0.875rem; box-sizing: border-box;`
- Readonly input: `opacity: 0.7; cursor: default;`
- `.claim-error`: `color: #e55; font-size: 0.8125rem; margin-top: 0.25rem;`
- Submit button: `margin-top: 1rem; padding: 0.5rem 1.25rem; background: var(--acc); color: #fff; border: none; border-radius: 4px; cursor: pointer; font-size: 0.875rem;`
- Button disabled: `opacity: 0.6; cursor: not-allowed;`
- `.claim-confirmation a`: `color: var(--acc); text-decoration: underline;`

Do NOT use `{:else}` inside the form — use two separate top-level `{#if}` blocks for form and confirmation states.
