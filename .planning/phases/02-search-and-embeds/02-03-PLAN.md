<task type="auto">
  <name>Task 1: Create search API endpoint and server load function</name>
  <files>src/routes/api/search/+server.ts, src/routes/search/+page.server.ts</files>
  <action>
    1. Create `src/routes/api/search/+server.ts`:
       - Export `GET` request handler
       - Read `q` (query string) and `mode` ('artist' | 'tag') from URL search params
       - Read optional `limit` param (default 50, max 100)
       - If `q` is empty, return empty array as JSON
       - Get `platform.env.DB` from the request event
       - If DB unavailable, return 503 with error message
       - Call `searchArtists()` or `searchByTag()` from `$lib/db/queries` based on mode
       - Return JSON response with `{ results, query, mode }` and `Cache-Control: public, max-age=60` header
       - Wrap in try/catch — return 500 with error message on failure

    2. Create `src/routes/search/+page.server.ts`:
       - Export `load` function (PageServerLoad)
       - Read `q` and `mode` from `url.searchParams`
       - If `q` is empty, return `{ results: [], query: '', mode: mode || 'artist' }`
       - Get DB from `platform.env.DB`
       - Call appropriate search function from `$lib/db/queries`
       - Return `{ results, query, mode }`
       - Handle DB unavailable gracefully (return empty results with error flag)
       - For tag mode: also return `matchedTag` so the UI can show "Showing artists tagged 'electronic'"
  </action>
  <verify>Run `npm run check` — no TypeScript errors. The search route loads successfully with `npm run dev` (will show empty results without D1, which is expected in local dev without wrangler).</verify>
  <done>Search API returns JSON results. Search page server load function queries D1 and passes results to the page component.</done>
</task>

       - Hover state: --bg-hover background, subtle border change
       - Card style: --bg-surface background, --border-subtle border, --card-radius border-radius
       - Information-dense: compact padding, no excessive whitespace
       - Keep cards same height within a row (use CSS grid with minmax)

    2. Create `src/routes/search/+page.svelte`:
       - Import `SearchBar` (normal size, pre-filled with current query and mode)
       - Import `ArtistCard`
       - Import `PROJECT_NAME` from config for page title
       - Page structure:
         a. SearchBar at the top (normal size), pre-filled with current query and mode from page data
         b. Results summary line: "{N} results for '{query}'" or "Showing artists tagged '{matchedTag}'" for tag mode
