<task type="auto">
  <name>Task 2: Create database query module and slug system</name>
  <files>src/lib/db/queries.ts, src/lib/utils/slug.ts, pipeline/add-slugs.js, pipeline/lib/schema.sql</files>
  <action>
    1. Create `src/lib/utils/slug.ts`:
       - Export `generateSlug(name: string): string` that lowercases, normalizes NFD, removes diacritics, replaces non-alphanumeric with hyphens, trims leading/trailing hyphens.
       - Keep it simple — regex-based, no external dependencies.

    2. Create `src/lib/db/queries.ts` with these exported functions:

       a. `sanitizeFtsQuery(input: string): string` — Remove FTS5 special characters (`'"(){}[]*:^~`) and reserved words (`AND`, `OR`, `NOT`, `NEAR`). Return cleaned string.

       b. `searchArtists(db: D1Database, query: string, limit?: number): Promise<ArtistResult[]>` — FTS5 prefix search (`clean*`) against `artists_fts`. Join to `artists` table to get mbid, name, country, type. Include top 5 tags via subquery on `artist_tags` (concatenated with `|` delimiter). Include a `slug` field. Fallback to LIKE query if FTS5 MATCH throws. Default limit: 50.

       c. `searchByTag(db: D1Database, tag: string, limit?: number): Promise<ArtistResult[]>` — Find artists by exact tag match in `artist_tags`. Join to `artists` for full artist data. Include the matched tag and its count as `matched_tag` and `match_count`. Include top 5 tags. Order by count descending. Default limit: 50.

       d. `getArtistBySlug(db: D1Database, slug: string): Promise<Artist | null>` — Look up artist by slug column. Return full artist record with all tags (not just top 5). Return null if not found.

       e. Define TypeScript interfaces: `ArtistResult` (for search results) and `Artist` (for full artist page data). Include `slug` field on both.

    3. Update `pipeline/lib/schema.sql`:
       - Add `slug TEXT` column to the `artists` table definition.
       - Add `CREATE INDEX IF NOT EXISTS idx_artists_slug ON artists(slug);`

    4. Create `pipeline/add-slugs.js`:
       - A Node.js script (using better-sqlite3, already in pipeline deps) that:
         a. Opens `pipeline/data/mercury.db`
         b. Adds `slug` column if it doesn't exist: `ALTER TABLE artists ADD COLUMN slug TEXT`
         c. Generates slugs for all artists using the same logic as `slug.ts`
         d. For duplicate slugs: append first 8 chars of MBID (e.g., `phoenix-a74b1b7f`)
         e. Updates all rows with their slugs
         f. Creates the slug index
         g. Reports stats: total artists, duplicates found, sample slugs
       - This script is run once on the existing database from Phase 1.

    5. Verify: `npm run check` passes. The query module imports cleanly. The pipeline script can be run with `node pipeline/add-slugs.js`.
  </action>
  <verify>Run `npm run check` — no TypeScript errors in queries.ts or slug.ts. Run `node pipeline/add-slugs.js` — script completes and reports slug statistics. Verify slug column exists in database.</verify>
  <done>Database query module exports searchArtists, searchByTag, getArtistBySlug with proper FTS5 sanitization and LIKE fallback. Slug column populated in database with collision handling. Pipeline schema updated.</done>
