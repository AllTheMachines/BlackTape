New tables to add:

```sql
CREATE TABLE IF NOT EXISTS detected_scenes (
    slug TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    tags TEXT NOT NULL,
    artist_mbids TEXT NOT NULL,
    listener_count INTEGER NOT NULL DEFAULT 0,
    is_emerging INTEGER NOT NULL DEFAULT 0,
    detected_at INTEGER NOT NULL
);

CREATE TABLE IF NOT EXISTS scene_follows (
    scene_slug TEXT PRIMARY KEY,
    followed_at INTEGER NOT NULL
);

CREATE TABLE IF NOT EXISTS scene_suggestions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    scene_slug TEXT NOT NULL,
    artist_mbid TEXT NOT NULL,
    artist_name TEXT NOT NULL,
    suggested_at INTEGER NOT NULL,
    UNIQUE(scene_slug, artist_mbid)
);

CREATE TABLE IF NOT EXISTS feature_requests (
    feature_id TEXT PRIMARY KEY,
    vote_count INTEGER NOT NULL DEFAULT 0,
    last_voted INTEGER NOT NULL
);
```

Then add the following Tauri command functions after the existing last command in the file. Follow the exact pattern used by existing commands (e.g., get_ai_setting): lock state.0.lock(), run SQL, return Result<T, String>.

Commands to implement:

1. `get_detected_scenes(state: TasteDbState) -> Result<Vec<DetectedSceneRow>, String>`
   - SELECT slug, name, tags, artist_mbids, listener_count, is_emerging, detected_at FROM detected_scenes ORDER BY detected_at DESC
   - Return Vec of a new struct DetectedSceneRow { slug, name, tags: String (raw JSON), artist_mbids: String (raw JSON), listener_count, is_emerging: bool, detected_at }

2. `save_detected_scenes(scenes: Vec<DetectedSceneRow>, state: TasteDbState) -> Result<(), String>`
   - DELETE FROM detected_scenes (full replace, not merge)
   - INSERT OR REPLACE for each scene in the input vec
   - Use unchecked_transaction() pattern (same as embedding store in embeddings.rs) to avoid double mutable borrow

3. `follow_scene(scene_slug: String, state: TasteDbState) -> Result<(), String>`
   - INSERT OR REPLACE INTO scene_follows (scene_slug, followed_at) VALUES (?1, ?2) with Unix timestamp

4. `unfollow_scene(scene_slug: String, state: TasteDbState) -> Result<(), String>`
   - DELETE FROM scene_follows WHERE scene_slug = ?1

5. `get_scene_follows(state: TasteDbState) -> Result<Vec<String>, String>`
   - SELECT scene_slug FROM scene_follows — returns list of followed slugs

6. `suggest_scene_artist(scene_slug: String, artist_mbid: String, artist_name: String, state: TasteDbState) -> Result<(), String>`
   - INSERT OR IGNORE INTO scene_suggestions (scene_slug, artist_mbid, artist_name, suggested_at) VALUES (...)

7. `get_scene_suggestions(scene_slug: String, state: TasteDbState) -> Result<Vec<SceneSuggestionRow>, String>`
   - SELECT artist_mbid, artist_name, suggested_at FROM scene_suggestions WHERE scene_slug = ?1 ORDER BY suggested_at ASC
   - New struct SceneSuggestionRow { artist_mbid, artist_name, suggested_at }

8. `upvote_feature_request(feature_id: String, state: TasteDbState) -> Result<i64, String>`
   - INSERT INTO feature_requests (feature_id, vote_count, last_voted) VALUES (?1, 1, ?2)
     ON CONFLICT(feature_id) DO UPDATE SET vote_count = vote_count + 1, last_voted = excluded.last_voted
   - Return the new vote_count via SELECT vote_count FROM feature_requests WHERE feature_id = ?1

Add the #[derive(Serialize, Deserialize, Debug)] structs (DetectedSceneRow, SceneSuggestionRow) at the top of the new section. Add `use std::time::{SystemTime, UNIX_EPOCH};` if not already imported for timestamp generation (or use `chrono::Utc::now().timestamp()` — check what existing commands use for timestamps).

Do NOT change any existing functions or table definitions.
