**In `src/routes/search/+page.ts`:**

Extend mode to handle `label` and `song` types explicitly:

```typescript
// Extended type support
type SearchType = 'artist' | 'tag' | 'label' | 'song';
let mode: SearchType = 'artist';

// Parse both mode and type params
const modeParam = url.searchParams.get('mode');
const typeParam = url.searchParams.get('type');

if (typeParam === 'label') mode = 'label';
else if (typeParam === 'song') mode = 'song';
else if (modeParam === 'tag') mode = 'tag';
else mode = 'artist';
```

Then update the query routing to handle `label` and `song` modes:
```typescript
if (mode === 'tag') {
  results = await searchByTag(provider, q);
} else if (mode === 'label') {
  results = await searchByLabel(provider, q);
} else if (intent.type === 'city') {
  results = await searchByCity(provider, intent.entity);
} else if (intent.type === 'label') {
  results = await searchByLabel(provider, intent.entity);
} else {
  results = await searchArtists(provider, q);
}

// For song mode, clear artist results and rely on localTracks section
if (mode === 'song') {
  results = [];
}
```

Also pass `mode` in the returned data:
```typescript
return { results, query: q, mode, matchedTag, intent, error: false, localTracks };
```
Update the return type annotation to `'artist' | 'tag' | 'label' | 'song'` where `'artist' | 'tag'` was.
