    **src/routes/api/curator-feature/+server.ts**
    - Pattern: fire-and-forget GET endpoint (same as existing API routes)
    - Read query params: `artist_param = url.searchParams.get('artist')` (MBID), `slug_param = url.searchParams.get('slug')`, `curator_handle = url.searchParams.get('curator')`, `source_param = url.searchParams.get('source') ?? 'embed'`
    - Validate: curator_handle must be present. Either `artist` (MBID) OR `slug` must be present. Return 400 if both missing.
    - Validate `curator_handle` is alphanumeric + hyphens/underscores/dots only (1-50 chars): `/^[\w\-\.]{1,50}$/`. Reject 400 if invalid — prevents XSS/injection from arbitrary handle values.
    - **MBID path:** if `artist` param provided, validate UUID format `/^[0-9a-f-]{36}$/`. Use directly.
    - **Slug path (for embed.js):** if `slug` param provided (no `artist`), validate slug: `/^[a-z0-9\-]{1,100}$/`. Look up MBID: `SELECT mbid FROM artists WHERE slug = ? LIMIT 1`. Return 400 if not found. Use the resolved MBID.
    - Rate limiting (simple): The same curator+artist pair should only be recorded once. Use `INSERT OR IGNORE` with UNIQUE constraint — `UNIQUE(artist_mbid, curator_handle)` in the table DDL. This naturally deduplicates.
    - INSERT: `INSERT OR IGNORE INTO curator_features (artist_mbid, curator_handle, featured_at, source) VALUES (?, ?, ?, ?)` with `Date.now()` as featured_at and `source_param` (validated: only allow 'embed' or 'collection', default 'embed')
    - Wrap in try/catch — return 200 even on DB error (fire-and-forget, don't break the blogger's page load)
    - Return `new Response('ok', { status: 200 })` — no JSON needed, this is a fire-and-forget ping
    - Add CORS header: `'Access-Control-Allow-Origin': '*'` — embed.js on external sites needs to call this endpoint cross-origin
    - Guard: if no `platform?.env?.DB`, return 200 silently

    Note: UNIQUE(artist_mbid, curator_handle) also handles the table DDL update — update the DDL in init-db.mjs to include this constraint.
