    CREATE TABLE IF NOT EXISTS curator_features (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      artist_mbid TEXT NOT NULL,
      curator_handle TEXT NOT NULL,
      featured_at INTEGER NOT NULL,
      source TEXT NOT NULL DEFAULT 'embed'
    );
    CREATE INDEX IF NOT EXISTS idx_cf_artist ON curator_features(artist_mbid);
    CREATE INDEX IF NOT EXISTS idx_cf_curator ON curator_features(curator_handle);

    **src/routes/api/curator-feature/+server.ts**
    - Pattern: fire-and-forget GET endpoint (same as existing API routes)
    - Read query params: `artist_mbid = url.searchParams.get('artist')`, `curator_handle = url.searchParams.get('curator')`
    - Validate: both must be non-empty strings. Return 400 if missing.
    - Validate `curator_handle` is alphanumeric + hyphens/underscores/dots only (1-50 chars): `/^[\w\-\.]{1,50}$/`. Reject 400 if invalid — prevents XSS/injection from arbitrary handle values.
    - Validate `artist_mbid` matches UUID format: `/^[0-9a-f-]{36}$/`. Reject 400 if invalid.
    - Rate limiting (simple): The same curator+artist pair should only be recorded once. Use `INSERT OR IGNORE` with UNIQUE constraint workaround — add a UNIQUE constraint: `UNIQUE(artist_mbid, curator_handle)` to the table DDL (add to init-db.mjs DDL above). This naturally deduplicates.
    - INSERT: `INSERT OR IGNORE INTO curator_features (artist_mbid, curator_handle, featured_at, source) VALUES (?, ?, ?, 'embed')` with `Date.now()` as featured_at
    - Wrap in try/catch — return 200 even on DB error (fire-and-forget, don't break the blogger's page load)
    - Return `new Response('ok', { status: 200 })` — no JSON needed, this is a fire-and-forget ping
    - Add CORS header: `'Access-Control-Allow-Origin': '*'` — embed.js on external sites needs to call this endpoint cross-origin
    - Guard: if no `platform?.env?.DB`, return 200 silently

    Note: UNIQUE(artist_mbid, curator_handle) also handles the table DDL update — update the DDL in init-db.mjs to include this constraint.
