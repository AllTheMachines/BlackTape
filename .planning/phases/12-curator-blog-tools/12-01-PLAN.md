    - Import `Feed` from `feed`, `getArtistBySlug` from `$lib/db/queries`, `D1Provider` from `$lib/db/d1-provider`
    - Format negotiation: `url.searchParams.get('format') === 'atom'` OR `(request.headers.get('Accept') ?? '').includes('application/atom+xml')`
    - Guard: if no `platform?.env?.DB`, return `new Response('Database not available', { status: 503 })`
    - Build Feed: `title: "${artist.name} — Mercury"`, `id: artistUrl`, `link: artistUrl`, `language: 'en'`, `copyright: 'Data from MusicBrainz (CC0)'`, `feedLinks: { rss: ..., atom: ...?format=atom }`
    - Add ONE item: the artist profile page entry with `title: "${artist.name} on Mercury"`, `id: artistUrl`, `link: artistUrl`, `description: "${artist.name}${country} — ${tags.slice(0,8).join(', ')}"`, `date: new Date()`
    - Return `new Response(xml, { headers: { 'Content-Type': contentType + '; charset=utf-8', 'Cache-Control': 'public, max-age=3600' } })`
    - Note: artist RSS is informational (current state snapshot), not event-driven — research confirmed this is correct for Phase 12

    **src/routes/api/rss/tag/[tag]/+server.ts**
    - Query artists with this tag: `SELECT a.id, a.mbid, a.name, a.slug, a.country, (SELECT GROUP_CONCAT(tag, ', ') FROM artist_tags WHERE artist_id = a.id LIMIT 8) AS tags FROM artists a JOIN artist_tags at2 ON at2.artist_id = a.id WHERE at2.tag = ? AND a.ended = 0 ORDER BY at2.count DESC LIMIT 50`
    - Feed: `title: "${decodedTag} — Mercury"`, `description: "Artists tagged ${decodedTag} on Mercury"`
    - Each artist becomes a feed item: `title: artist.name`, `link: ${siteUrl}/artist/${artist.slug}`, `description: artist country + tags`
    - Tag param comes URL-decoded from params.tag (SvelteKit handles this)
    - 404 if no artists found for tag

    **src/routes/api/rss/curator/[handle]/+server.ts**

    - Wrap the curator_features query in try/catch — table may not exist yet (created in Plan 03). Return empty feed with descriptive message if table missing.
    - Feed: `title: "${handle}'s Discoveries — Mercury"`, each artist is a feed item
