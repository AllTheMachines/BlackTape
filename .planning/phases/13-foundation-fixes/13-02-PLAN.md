  <action>
**StyleMap.svelte** — The container div is at line 93: `<div class="style-map-container" bind:this={container}>`. The `{#if layoutNodes.length === 0}` block is inside. The `data-ready` attribute should be on the outer container div so it is always present in the DOM before simulation runs:

Change:
```svelte
<div class="style-map-container" bind:this={container}>
```
To:
```svelte
<div class="style-map-container" bind:this={container}
     data-ready={layoutNodes.length > 0 ? 'true' : undefined}>
```

This works because: `layoutNodes` is a `$state` variable. After `layoutNodes = settled` is assigned inside `onMount` (line 85), Svelte 5 reactively sets the attribute. The container div is unconditionally rendered (the `{#if}` is inside it), so the attribute is always reachable by `waitForSelector`.

**GenreGraph.svelte** — Find the component's root container `<div>` that wraps the SVG. Read the full component to find the line. Apply the same pattern: add `data-ready={layoutNodes.length > 0 ? 'true' : undefined}` to the container div. The `layoutNodes` state variable is assigned after `simulation.tick(300)` in GenreGraph's `onMount`. Verify the exact variable name by reading the component file first.

**TasteFingerprint.svelte** — Same pattern. Read the component to find the root container div and the state variable name assigned after `simulation.tick()`. Add `data-ready={<layoutVariable>.length > 0 ? 'true' : undefined}` to the container div.

**Important for all three:** The `data-ready` attribute is on the OUTERMOST component container div, not on the SVG element or any inner element. This ensures Playwright can find it with `waitForSelector('[data-ready="true"]')` regardless of whether the SVG is rendered yet.

**Pitfall:** If a component has conditional rendering around the container (e.g., `{#if nodes.length > 0}...`), and the outer wrapping element is in the parent page, put `data-ready` on the closest always-rendered ancestor inside the component. Check the actual structure before editing.
