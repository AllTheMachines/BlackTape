  <action>
Create three new files in `src/lib/theme/`:

**palette.ts** — Pure functions, no side effects:
- `tasteTagsToHue(tags: TasteTag[]): number` — Takes the user's taste tags (import TasteTag type from `$lib/taste/profile.svelte`), filters to weight > 0, sorts by weight descending, takes top 5, sorts alphabetically by tag name, joins with '|', hashes to 0-360 hue. Use djb2-style hash: `hash = ((hash << 5) - hash + charCode) | 0`, then `((hash % 360) + 360) % 360`.
- `generatePalette(hue: number): Record<string, string>` — Returns a map of CSS custom property names to OKLCH values. Generate ALL taste-affected properties from a single hue. Properties to generate (matching existing theme.css names):
  - `--bg-base`: `oklch(0.05 0.01 ${hue})`
  - `--bg-surface`: `oklch(0.10 0.015 ${hue})`
  - `--bg-elevated`: `oklch(0.14 0.02 ${hue})`
  - `--bg-hover`: `oklch(0.18 0.025 ${hue})`
  - `--border-subtle`: `oklch(0.12 0.015 ${hue})`
  - `--border-default`: `oklch(0.17 0.02 ${hue})`
  - `--border-hover`: `oklch(0.22 0.025 ${hue})`
  - `--link-color`: `oklch(0.72 0.12 ${hue})`
  - `--tag-bg`: `oklch(0.13 0.04 ${hue})`
  - `--tag-text`: `oklch(0.72 0.12 ${hue})`
  - `--tag-border`: `oklch(0.22 0.05 ${hue})`
  - `--progress-color`: `oklch(0.72 0.12 ${hue})`
  - `--player-bg`: `oklch(0.04 0.01 ${hue})`
  - `--player-border`: `oklch(0.10 0.015 ${hue})`
  Keep --text-primary, --text-secondary, --text-muted, --text-accent at FIXED lightness (not hue-tinted) for WCAG AA readability. Do NOT include them in the generated palette.
- Export `TASTE_PALETTE_KEYS: string[]` — array of all property names the engine manages, used by clearPalette.

**engine.svelte.ts** (uses .svelte.ts for $state rune):
- Export `themeState = $state({ mode: 'default' as 'taste' | 'manual' | 'default', manualHue: 220, computedHue: 0, palette: {} as Record<string, string> })`.
- `applyPalette(palette: Record<string, string>): void` — Sets each entry on `document.documentElement.style`. Add `transition: background-color 0.5s, border-color 0.5s, color 0.5s` to root before applying, then remove the transition after 600ms timeout (smooth fade per user decision, avoids permanent transition performance cost).
- `clearPalette(): void` — Removes all TASTE_PALETTE_KEYS from document.documentElement.style. Reverts to theme.css defaults.
- `initTheme(tags: TasteTag[], savedPrefs: { mode: string; manualHue: string }): void` — Called on app startup. If mode='taste' and tags exist, compute hue from tags, generate palette, apply. If mode='manual', use saved manualHue. If mode='default', do nothing (theme.css defaults). Update themeState reactively.
- `updateThemeFromTaste(tags: TasteTag[]): void` — Called when taste profile changes. Only acts if mode='taste'. Recomputes hue, generates palette, applies.

**preferences.ts** — Preference CRUD using existing taste.db pattern:
- `async getInvoke()` — same dynamic import pattern as ai/state.svelte.ts.
- `async loadThemePreferences(): Promise<{ mode: string; manualHue: string }>` — Reads `theme_mode` and `theme_manual_hue` from ai_settings via `get_all_ai_settings`. Returns defaults ('default', '220') if not set.
- `async saveThemePreference(key: 'theme_mode' | 'theme_manual_hue', value: string): Promise<void>` — Writes via `set_ai_setting`.
- `async loadStreamingPreference(): Promise<string>` — Reads `preferred_platform` from ai_settings. Returns '' if not set.
- `async saveStreamingPreference(platform: string): Promise<void>` — Writes `preferred_platform`.
- `async loadLayoutPreference(): Promise<string>` — Reads `layout_template`. Returns 'cockpit' if not set (cockpit is default per user decision).
- `async saveLayoutPreference(template: string): Promise<void>` — Writes `layout_template`.

All preference functions follow the exact same pattern as `loadAiSettings`/`saveAiSetting` in `src/lib/ai/state.svelte.ts`. No new Rust commands needed — the existing `get_all_ai_settings` and `set_ai_setting` commands handle arbitrary key-value pairs.
  </action>
