```markdown
### Underground Aesthetic

The theme engine and panel layout system that makes Mercury feel like a place, not a tool.

**Theme Engine** (`src/lib/theme/`)
- `palette.ts` — OKLCH palette generation from taste tags. `tasteTagsToHue()` hashes top-5 weighted tags into a deterministic 0-360 hue. `generatePalette()` produces CSS custom property overrides at fixed lightness/chroma curves.
- `engine.svelte.ts` — Reactive theme state via `$state` rune. `computeAndApplyTheme()` applies palette to `document.documentElement.style`. Three modes: default (neutral), taste (auto from profile), manual (user-picked hue).
- `preferences.ts` — Preference CRUD via existing `taste.db` `ai_settings` table. Stores theme_mode, theme_manual_hue, layout_template, preferred_platform. Same invoke pattern as AI settings.
- `templates.ts` — Layout template definitions. Four templates: default (single column), cockpit (3-pane), focus (2-pane), minimal (full-width overlay).

**Panel Layout** (`src/lib/components/PanelLayout.svelte`)
- PaneForge-based resizable panel layout. Tauri-only — web stays single-column.
- Templates define pane arrangement; PaneForge `autoSaveId` persists sizes per-template in localStorage.
- Snippets: `sidebar` (left), `children` (main), `context` (right).

**Interactive Controls** (`src/lib/components/ControlBar.svelte`)
- Dense toolbar with tag filter, view toggle, sort dropdown.
- Used in sidebar panels and inline above content.

**Streaming Preference**
- User sets preferred platform in Settings → stored in taste.db.
- `EmbedPlayer.svelte` and artist page "Listen On" bar reorder platforms: preferred first, then default order.
- Server data stays neutral; client-side sorting only.

**Anti-Patterns (Phase 8)**
| Pattern | Problem | Use Instead |
|---------|---------|-------------|
| HSL palette generation | Perceptually non-uniform — equal L values look different across hues | OKLCH with fixed lightness curves |
| Theme in localStorage | Splits source of truth (taste.db for some, localStorage for others) | taste.db ai_settings for preferences; PaneForge autoSaveId for panel sizes only |
| PanelLayout on web | No user state on web, panels without content are empty noise | isTauri() gate — web stays single-column |
| Partial palette override | Some vars tinted, others not → visual inconsistency | Apply ALL palette properties or NONE |
```

```markdown
### Appearance

Mercury can match its colors to your musical taste.

**Theme Modes:**
- **Default** — Neutral dark theme with blue accents (the classic look)
- **Taste** — Colors generated from your taste profile. Your top tags are hashed into a unique accent color. Two different listeners see two different Mercurys.
- **Manual** — Pick your own accent color using the hue slider (0-360 degrees)

Go to Settings → Appearance to choose your mode. Changes apply instantly.

### Layout

Choose how panels are arranged in your workspace:

- **Default** — Full-width single column (the original layout)
- **Cockpit** — Three resizable panels: sidebar, main content, and context. Drag the dividers to resize.
- **Focus** — Two panels: main content with a context sidebar
- **Minimal** — Full-width with compact controls

Go to Settings → Layout to switch. Panel sizes are remembered between sessions.

### Streaming Preference

Set your preferred streaming platform:

- **None** — Default order (Bandcamp → Spotify → SoundCloud → YouTube)
- **Bandcamp / Spotify / SoundCloud / YouTube** — Your choice appears first in embeds and "Listen on" links

Go to Settings → Streaming Preference to set.
```
