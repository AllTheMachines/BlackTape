export type LayoutTemplate = 'cockpit' | 'focus' | 'minimal';

export interface TemplateConfig {
  id: LayoutTemplate;
  label: string;
  description: string;
  panes: 'three' | 'two' | 'one';
  /** PaneForge autoSaveId — unique per template so sizes persist independently */
  autoSaveId: string;
  leftDefault: number;   // default % width for left pane
  mainDefault: number;   // default % width for main pane
  rightDefault: number;  // default % width for right pane (0 = not shown)
  leftMin: number;       // minimum % width for left pane
  mainMin: number;       // minimum % width for main pane
  rightMin: number;      // minimum % for right pane
}

    mainMin: 35,

**PanelLayout.svelte** — The master layout wrapper. Renders different PaneForge arrangements based on the active template.

Props: `template: LayoutTemplate`, `sidebar: Snippet` (left sidebar content), `context: Snippet` (right sidebar content), `children: Snippet` (main content).

Structure:
- Import `PaneGroup, Pane, PaneResizer` from `paneforge`.
- Import `LAYOUT_TEMPLATES` from `$lib/theme/templates`.
- Get config from `LAYOUT_TEMPLATES[template]`.
- Conditional rendering based on `config.panes`:
  - `'three'`: PaneGroup(horizontal) with left Pane + PaneResizer + main Pane + PaneResizer + right Pane. Use `autoSaveId={config.autoSaveId}`. Set `defaultSize` and `minSize` from config.
  - `'two'`: PaneGroup(horizontal) with main Pane + PaneResizer + right Pane. Use `autoSaveId={config.autoSaveId}`.
  - `'one'`: No PaneGroup — just render `{@render children()}` directly.
- For the left pane, render the `sidebar` snippet. For right pane, render the `context` snippet. For main pane, render `children`.
- Add `collapsible={true}` on left and right Panes. When collapsed, show a thin strip (24px) with an expand button (arrow icon).
- PaneResizer styling: 4px wide, hover color var(--border-hover), cursor col-resize. Style with class `pane-resizer`.
- Wrap the entire layout in a `<div class="panel-layout">` with `height: calc(100vh - var(--header-height))` (fill viewport below header). If player is active (accept `hasPlayer: boolean` prop), subtract player height too: `height: calc(100vh - var(--header-height) - var(--player-height))`.

**RightSidebar.svelte** — Context-aware content panel.

Props: `pagePath: string` (current route path from $page), `artistData: any` (optional, passed when on artist page), `genreData: any` (optional, passed when on genre page).

Sections (conditionally rendered based on pagePath):
1. **Artist context** (when pagePath starts with `/artist/`): Show "Related" header + list of related artist tags from artistData. Show recommendation section if AI ready. Show queue preview (first 3 items).
2. **Genre context** (when pagePath starts with `/kb/genre/`): Show subgenres, related genres, key artists list.
3. **Default context** (all other pages): Show "Now Playing" info if a track is playing (from playerState), queue preview, and "Your Taste" summary (top 5 taste tags from tasteProfile).

Style: same bg/padding/overflow pattern as LeftSidebar. Sections separated by `var(--border-subtle)` lines.

**ControlBar.svelte** — Dense toolbar that sits above the panel layout.

Props: `currentTemplate: LayoutTemplate`, `onTemplateChange: (template: LayoutTemplate) => void`.

Structure (all in a single 32px-high bar):
- **Left group:** Search icon + compact search input (200px wide, type-to-search). On submit, `goto('/search?q=...')`.
- **Center group:** Empty — reserved for future controls (keeps the bar balanced).
- **Right group:**

  - Theme indicator: Small colored circle showing current theme hue (filled with `oklch(0.72 0.12 ${hue})`). Clicking navigates to settings page.

Style: `background: var(--bg-surface)`, `border-bottom: 1px solid var(--border-subtle)`, `height: 32px`, `display: flex`, `align-items: center`, `padding: 0 var(--space-sm)`, `gap: var(--space-sm)`. Inputs and buttons use small sizing (0.75rem font, compact padding).
