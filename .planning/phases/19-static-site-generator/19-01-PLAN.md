**Helper functions (private, not pub):**

`html_escape(s: &str) -> String` — inline escape, no crate:
```rust
fn html_escape(s: &str) -> String {
    s.replace('&', "&amp;")
     .replace('<', "&lt;")
     .replace('>', "&gt;")
     .replace('"', "&quot;")
     .replace('\'', "&#x27;")
}
```

`download_cover(url: &str, dest: &std::path::Path) -> bool` — async, best-effort, returns false on any error:
```rust
async fn download_cover(url: &str, dest: &std::path::Path) -> bool {
    let Ok(response) = reqwest::get(url).await else { return false; };
    if !response.status().is_success() { return false; }
    let Ok(bytes) = response.bytes().await else { return false; };
    std::fs::write(dest, &bytes).is_ok()
}
```

`build_tags_html(tags: &str) -> String` — splits comma-separated tags string, wraps each in `<span class="tag">`, returns empty string if no tags.

`build_releases_html(releases: &[ReleaseSitePayload], downloaded: &HashSet<String>) -> String` — for each release, builds a `<div class="release-card">` with:
- If `downloaded.contains(&release.mbid)`: `<img src="covers/{mbid}.jpg" alt="{title}" class="cover" loading="lazy">`
- Else: an inline SVG placeholder `<div class="cover-placeholder"><svg ...></svg></div>` (a simple gray rectangle with a music note symbol: `<svg width="120" height="120" viewBox="0 0 120 120" xmlns="http://www.w3.org/2000/svg"><rect width="120" height="120" fill="#1c1c1c" rx="4"/><text x="60" y="70" font-size="40" text-anchor="middle" fill="#404040">♪</text></svg>`)
- `<div class="release-info">`: title (escaped), year if present, type (escaped)
- `<div class="release-links">`: for each link, `<a href="{url}" target="_blank" rel="noopener noreferrer" class="link-{platform}">{platform_label}</a>`

Platform labels mapping (lowercase platform string → display label): `"bandcamp" → "Bandcamp"`, `"spotify" → "Spotify"`, `"soundcloud" → "SoundCloud"`, `"youtube" → "YouTube"`, anything else → capitalize first letter.

`build_html(artist: &ArtistSitePayload, downloaded: &HashSet<String>) -> String` — builds the entire HTML document using `format!(r#"..."#, ...)` with inline `<style>` block. Use hex/RGB colors only (NOT OKLCH — the generated site is opened in users' various browsers). CSS must be responsive single-column centered layout with Mercury dark theme appearance. All text fields (name, bio, tags, release titles, type, country) must go through `html_escape()`. URLs in href attributes must NOT go through `html_escape()` but MUST be in quoted attributes. Footer: `<footer><small>Made with <a href="https://github.com/yourusername/mercury">Mercury</a></small></footer>`. Include these CSS custom properties in the inline style block (static hex translations):
```css
--bg-base: #111111;
```

**Commands:**

`generate_artist_site` (public async, #[tauri::command]):
```
pub async fn generate_artist_site(output_dir: String, artist: ArtistSitePayload) -> Result<SiteGenResult, String>
```
Steps:
1. Build output path: `PathBuf::from(&output_dir).join(&artist.slug)`
2. Build covers path: `out_path.join("covers")`
3. `std::fs::create_dir_all(&covers_path).map_err(|e| format!("Failed to create output dir: {}", e))?`
4. Loop over `artist.releases` sequentially (NOT parallel — avoid rate-limiting Cover Art Archive). For each with a `cover_art_url`, call `download_cover(url, &covers_path.join(format!("{}.jpg", release.mbid))).await`. Insert `release.mbid.clone()` into `downloaded_mbids: HashSet<String>` on success.
5. Call `build_html(&artist, &downloaded_mbids)` to get the HTML string.
6. Write to `out_path.join("index.html")` using `std::fs::write`. Map error with `format!("Failed to write index.html: {}", e)`.
7. Return `Ok(SiteGenResult { output_dir: out_path.to_string_lossy().to_string(), cover_count: downloaded_mbids.len() })`
