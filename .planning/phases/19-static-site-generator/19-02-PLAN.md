**Props (Svelte 5 $props()):**
```typescript
let {
  artist,    // { name: string; slug: string; tags: string | null; mbid: string }
  releases,  // Array<{ mbid: string; title: string; year: number | null; type: string; coverArtUrl: string | null; links: Array<{ url: string; platform: string }> }>
  bio,       // string | null — effectiveBio from parent
  onclose,   // () => void — parent closes dialog
}: {
  artist: { name: string; slug: string; tags: string | null; mbid: string };
  releases: Array<{ mbid: string; title: string; year: number | null; type: string; coverArtUrl: string | null; links: Array<{ url: string; platform: string }> }>;
  bio: string | null;
  onclose: () => void;
} = $props();
```

**State machine ($state variables):**
```typescript
type DialogState = 'confirming' | 'picking' | 'generating' | 'success' | 'error';
let dialogState = $state<DialogState>('confirming');
let errorMessage = $state('');
let generationResult = $state<{ output_dir: string; cover_count: number } | null>(null);
```

**Derived display values:**
```typescript
let tagList = $derived(
  artist.tags ? artist.tags.split(', ').filter(Boolean) : []
);
```

**Main action — handleConfirm (async):**
1. Set `dialogState = 'picking'`
2. Import and call `open` from `@tauri-apps/plugin-dialog`: `const selected = await open({ directory: true, multiple: false, title: 'Choose export folder' })`
3. If `!selected` (user cancelled): set `dialogState = 'confirming'` and return early
4. Set `dialogState = 'generating'`
5. Import `invoke` from `@tauri-apps/api/core`
6. Build the artist payload — map releases to the Rust struct shape:
```typescript
const artistPayload = {
  name: artist.name,
  slug: artist.slug,
  tags: artist.tags ?? '',
  country: null,        // Not available on this component — pass null
  type: null,           // Not available on this component — pass null
  begin_year: null,     // Not available on this component — pass null
  ended: false,
  bio: bio ?? null,
  releases: releases.map(r => ({
    mbid: r.mbid,
    title: r.title,
    year: r.year,
    type: r.type,
    cover_art_url: r.coverArtUrl ?? null,
    links: r.links.map(l => ({ url: l.url, platform: l.platform })),
  })),
};
```
7. Call `const result = await invoke<{ output_dir: string; cover_count: number }>('generate_artist_site', { outputDir: selected, artist: artistPayload })`
8. On success: `generationResult = result; dialogState = 'success'`
9. On error (catch): `errorMessage = String(e); dialogState = 'error'`

**handleOpenFolder:**
```typescript
async function handleOpenFolder() {
  if (!generationResult) return;
  const { invoke } = await import('@tauri-apps/api/core');
  await invoke('open_in_explorer', { path: generationResult.output_dir }).catch(() => {});
}
```

**Template structure:**

The dialog is a fixed-position overlay (modal). Use a backdrop div + centered content card. Mercury dark theme styles inlined in `<style>` block. Dismiss on backdrop click (call `onclose()`), not on content click.

State machine rendering:

**confirming state:** Show "Export site for {artist.name}" heading. Show a summary list:
- "Bio: {bio ? 'included' : 'not available'}"
- "{releases.length} releases"
- "{tagList.length} tags"

Note text (smaller): "Cover art will be downloaded at generation time. Missing covers will be replaced with a placeholder."

Two buttons: "Export site" (primary, calls `handleConfirm`) and "Cancel" (secondary, calls `onclose`).

**picking state:** Show spinner text "Choose a folder…" — no buttons.

**generating state:** Show spinner text "Generating site… downloading covers." — no buttons. Note: cover downloads can take a few seconds for artists with many releases.

**success state:** Show checkmark heading "Site exported!"
- Show path: `<code>{generationResult?.output_dir}</code>`
- Show: "{generationResult?.cover_count} cover image{generationResult?.cover_count === 1 ? '' : 's'} downloaded."
- Two buttons: "Open folder" (calls `handleOpenFolder`) and "Done" (calls `onclose`).

**error state:** Show "Export failed" heading. Show `<pre class="error-text">{errorMessage}</pre>`. One button: "Close" (calls `onclose`).

**Styling:** Mercury dark theme. Backdrop: rgba(0,0,0,0.7). Card: background #1c1c1c, border 1px solid #333, border-radius 8px, padding 1.5rem, max-width 480px. Buttons follow project conventions (primary = solid, secondary = outline). Spinner is a simple CSS animation (rotating border). Inline all styles in `<style scoped>` — no external CSS imports.
