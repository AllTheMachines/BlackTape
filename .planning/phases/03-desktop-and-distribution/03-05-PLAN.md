Configure the Tauri auto-updater with signing keys and produce a working app installer.

Purpose: The updater requires cryptographic signing -- the private key cannot be regenerated later. This plan generates the key pair early (critical -- losing the private key means no future updates), configures the updater endpoint in tauri.conf.json, and verifies the app can be built into an installer. The endpoint can be a placeholder URL for now (actual update server comes later). The important thing is that the signing infrastructure exists from day one.

Output: Updater configuration, signing key pair, working app build producing an installer.
</objective>

  <action>
    **Step 1: Generate the updater signing key pair**

    Run: `npx tauri signer generate -w ~/.tauri/mercury.key`

    This generates:
    - Private key: `~/.tauri/mercury.key` (MUST be kept secret, backed up securely)
    - Public key: displayed in stdout (goes into tauri.conf.json)
    - Password: if prompted, set one and record it (needed for builds)

    IMPORTANT: The private key file MUST NOT be committed to git. Add `*.key` to .gitignore if not already there. Store the password securely.

    **Step 2: Configure updater in tauri.conf.json**

    Add updater configuration under the `plugins` section:
    ```json
    "updater": {
      "active": true,
      "dialog": true,
      "pubkey": "<paste the public key from Step 1>",
      "endpoints": [
        "https://mercury-updates.example.com/{{target}}/{{arch}}/{{current_version}}"
      ]
    }
    ```

    Notes:
    - `dialog: true` shows a built-in update dialog (simplest UX for now)
    - The endpoint URL is a placeholder -- replace with real URL when update server is set up
    - The `{{target}}`, `{{arch}}`, `{{current_version}}` are Tauri template variables

    **Step 3: Set app version**

    In tauri.conf.json, set `version` to "0.1.0" (first desktop release).

    **Step 4: Configure NSIS installer (Windows)**

    In tauri.conf.json under `bundle`:
    ```json
    "bundle": {
      "active": true,
      "targets": ["nsis"],
      "icon": ["icons/32x32.png", "icons/128x128.png", "icons/icon.ico"],
      "identifier": "com.mercury.app",
      "windows": {
        "nsis": {
          "installMode": "currentUser"
        }
      }
    }
    ```

    Use NSIS (not MSI) because NSIS handles WebView2 bootstrapping automatically on Windows 10.

    **Step 5: Create placeholder icons**

    If icons don't exist in `src-tauri/icons/`, create minimal placeholder icons. Tauri needs at least `icon.ico` (Windows) and `icon.png`. Use a simple generated icon or copy from Tauri's default scaffolding. The real icons come later when the project has branding.
  </action>

    4. The private key is NOT in the git repository (check with `git status`).
  </verify>
