2. **Replace ArtistCard with a match-aware row in the discovery section** — instead of `<ArtistCard>`, render each result with a match badge. The badge uses `artist.match_type` from the query result (set by searchByCity/searchByLabel as a literal SQL alias, or determined from data.mode for existing searches).

The match type logic:
- If `data.intent.type === 'city'` → badge = "City match"
- If `data.intent.type === 'label'` → badge = "Label match"
- If `data.mode === 'tag'` → badge = "Tag match"
- Otherwise → badge = "Name match"

Keep using `<ArtistCard>` for the card rendering itself, but pass the `matchReason` prop with the correct badge text:

```svelte
{#each data.results as artist}
  <ArtistCard
    {artist}
    matchReason={
      data.intent?.type === 'city' ? 'City match' :
      data.intent?.type === 'label' ? 'Label match' :
      data.mode === 'tag' ? `Tag match: ${data.matchedTag}` :
      'Name match'
    }
    data-testid="search-result-card"
  />
{/each}
```

Note: ArtistCard already renders `matchReason` as `.match-reason` text — this satisfies the badge display requirement. The "highlight matched tag" visual requirement for tag match rows is satisfied by passing the matched tag name in the matchReason string.
