  <files>src/lib/player/queue.svelte.ts</files>
  <action>
Extend queue.svelte.ts with two capabilities:

**1. localStorage persistence**

Add two internal functions (not exported):
- `saveQueueToStorage()`: serializes `queueState.tracks` and `queueState.currentIndex` to `localStorage.setItem('mercury:queue', JSON.stringify({tracks, currentIndex}))`. Call this after every mutation (setQueue, addToQueue, addToQueueNext, removeFromQueue, clearQueue).
- `loadQueueFromStorage()`: reads `localStorage.getItem('mercury:queue')`, parses the JSON, sets `queueState.tracks` and `queueState.currentIndex` from it. If the stored index points to a valid track, set `playerState.currentTrack` to that track but do NOT auto-play (preserve isPlaying=false). Wrap in try/catch — if corrupt, ignore.

Export a new function `restoreQueueFromStorage(): void` that calls `loadQueueFromStorage()`. This will be called from the root layout on mount.

**2. Play behavior: insert-next (not replace)**

The CONTEXT.md specifies: "Play mid-queue: inserts the clicked track as the next track (after current), queue resumes — does NOT interrupt or replace."

The existing `setQueue()` replaces the entire queue. That's correct for "play from beginning" (empty queue or explicit "Play Album"). No changes needed there.

Add a new exported function:
```ts
export function playNextInQueue(track: PlayerTrack): void
```
This function: inserts `track` at `currentIndex + 1` (using the same splice logic as `addToQueueNext`), then immediately plays it by calling `playTrack(track)` and setting `queueState.currentIndex = queueState.currentIndex + 1`. This is used when the user clicks the play icon on a track row and there is already something playing.

Also export a helper to determine if anything is currently playing:
```ts
export function isQueueActive(): boolean {
  return queueState.tracks.length > 0 && playerState.isPlaying;
}
```

**Play decision logic** (used by TrackRow in Plan 02): if `isQueueActive()` is true, call `playNextInQueue(track)`. If not active, call `setQueue(contextTracks ?? [track], 0)`. TrackRow will handle this — queue.svelte.ts just needs to export the primitives.

Also add `reorderQueue(from: number, to: number): void` (used by Queue.svelte for drag-reorder in Plan 03):
```ts
export function reorderQueue(from: number, to: number): void {
  const newTracks = [...queueState.tracks];
  const [moved] = newTracks.splice(from, 1);
  newTracks.splice(to, 0, moved);
  queueState.tracks = newTracks;
  if (queueState.currentIndex === from) {
    queueState.currentIndex = to;
  } else if (from < queueState.currentIndex && to >= queueState.currentIndex) {
    queueState.currentIndex--;
  } else if (from > queueState.currentIndex && to <= queueState.currentIndex) {
    queueState.currentIndex++;
  }
  saveQueueToStorage();
}
```
