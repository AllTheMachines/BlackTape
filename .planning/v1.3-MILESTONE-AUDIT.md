### Integration Issues

#### Non-Critical: Nostr initialization order (soft concern â€” ROOM-01 through ROOM-05)

`initNostr()` is not called at room page or scene page `onMount`. Both routes depend on `ndkState.ndk` being non-null.

**Behavior breakdown:**
- Scene page `checkActiveRoom`: null-guarded â€” returns `{active: false}` silently. Graceful degradation. User sees "Start listening room" when there may be an active room.
- Room page `joinRoom`: throws `'NDK not initialized'` if NDK not yet initialized. Room page `onMount` does not catch this throw. Result: shows "No active room for this scene" even if a room exists. Wrong state, non-crash.

**In practice:** NDK is initialized when users visit `/backers`, use DMs (Phase 10), or trigger other Nostr features. Edge case only affects users who navigate directly to a room/scene page on a cold app start without having used other Nostr features first.

**Fix:** Add `initNostr()` call at room page `onMount` before `checkActiveRoom`/`joinRoom`.

#### Cosmetic: Dead import

`openRoom` imported in `scenes/[slug]/+page.svelte` line 8 but never called. `handleStartRoom()` correctly uses `goto('/room/' + slug)` â€” room creation happens inside the room page itself. No functional impact.
