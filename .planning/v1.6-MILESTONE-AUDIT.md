### Integration Gaps (Forward-Phase Risks)

**Gap 1 — serviceOrder vs streamingPref.platform (two ordering systems)**
- `streamingState.serviceOrder` (Phase 29): ranked list, persisted to db, shown in Settings drag-UI
- `streamingPref.platform` (legacy): single-platform preference, used by `EmbedPlayer.orderedPlatforms`
- Phase 31 must migrate EmbedPlayer to consume `serviceOrder` for auto-load-first-preferred behavior
- Affects: INFRA-01 (end-to-end), PLAYER-02

**Gap 2 — SC-01 two code paths**
- `EmbedPlayer.svelte` accepts `soundcloudEmbedHtml` prop but artist page doesn't pass it
- Artist page uses a separate `inlinePlayerHtml` / `onplayinline` path (lines ~541+) for SC embeds
- Widget API PLAY coordination hooks exist only in `EmbedPlayer` path — not in `inlinePlayerHtml` path
- Phase 31 must pick one canonical SC embed path and ensure coordination (SC-02) works through it
- Affects: SC-01, SC-02

**Gap 3 — No activeSource watcher to unmount competing embeds**
- When switching sources in Phase 31, the old embed must be unmounted (not just paused)
- No `$effect` watching `streamingState.activeSource` exists to trigger unmount
- Phase 31 must add this reactive behavior (unmount-on-switch)
- Affects: SC-02, YT-01, BC-01, PLAYER-02

**Gap 4 — Spotify postMessage schema unverified**
- `EmbedPlayer.svelte:46`: "Known Spotify embed event types as of Feb 2026 — verify in dev"
- Types (`playback_update`, `player_state_changed`) are from community reports
- Must be verified against live embed before Phase 30 builds Spotify Connect on top
- Affects: SPOT-02 (if detection is wrong, Spotify embed play won't trigger coordination)

**Gap 5 — PLAYER-01 via-badge conditional on local track**
- Player bar only renders when `playerState.currentTrack !== null`
- Via-badge lives inside this conditional — invisible in embed-only sessions
- Phase 31 source-switcher (PLAYER-02) on artist page serves as the primary source indicator during embed sessions
- Low severity: acceptable UX tradeoff per Phase 29 scope decision

**Gap 6 — No release route for Phase 32**
- `/artist/[slug]/release/[mbid]` is referenced in `ReleaseCard.svelte` but the route directory doesn't exist
- Phase 32 must create this route before implementing BC-02 and PLAYER-03
- `release.links` typing in `types.ts` is sufficient (no data plumbing needed)
- Affects: BC-02, PLAYER-03

## Tech Debt

**Phase 29 — Streaming Foundation**
- `PLAYER-01` via-badge scoped inside `{#if playerState.currentTrack}` — invisible during embed-only sessions. Intentional scope decision; Phase 31 PLAYER-02 (artist-page source-switcher) is the designed mitigation.
- Spotify postMessage event types in `EmbedPlayer.svelte:46` are community-sourced and unverified. Must be confirmed in Phase 30.
- Two overlapping ordering systems (`streamingPref.platform` single-select + `streamingState.serviceOrder` ranked list). Phase 31 must consolidate to avoid split behavior.
