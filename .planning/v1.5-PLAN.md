## Phase 29: Spotify Full Integration ⭐ Priority

Full Spotify playback inside BlackTape. Not an embed — a real connected account experience.

### What this means
- User connects their Spotify account (OAuth)
- Any track BlackTape surfaces can be played directly via Spotify
- Full playback controls (play/pause/skip/seek) in the BlackTape player
- Spotify Premium required on user's end (this is Spotify's restriction, not ours)
- Free tier: can still browse/search Spotify, just can't play full tracks

### Work
- Spotify Developer app registration (Client ID, redirect URI)
- OAuth 2.0 authorization code flow with PKCE
- Tauri command `spotify_authorize()` → opens system browser auth popup
- Token storage in Tauri secure store (refresh token persisted)
- `src-tauri/src/streaming/spotify.rs` — Spotify Web API client
  - `search(query)` — find tracks matching an artist/release
  - `get_tracks_for_artist(artist_name)` — top tracks
- Spotify Web Playback SDK wired into the BlackTape player
  - SDK loads in WebView, initializes device
  - `play_track(track_uri)` — plays directly in-app
  - Player state syncs with BlackTape player bar (play/pause/progress/title)
- Artist page: "Play on Spotify" becomes "Play" when connected
- Release page: full album playable track-by-track
- Graceful degradation: if not connected, show "Connect Spotify" prompt

### Acceptance
- User can connect Spotify account in Settings
- Connected badge shown in player bar
- Any artist/release page: tracks playable via Spotify
- Player controls work (play/pause/seek/next)
- Token refreshes automatically without re-auth
