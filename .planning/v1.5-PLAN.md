## Phase 29: Streaming API Integration

Add Spotify, YouTube, SoundCloud, Bandcamp playback to Mercury's player.

### 29.1 — Spotify Authentication & Playback

**Work:**
- Spotify Developer app registration + OAuth setup
- `lib/spotify.ts` — Client credentials or authorization code flow (decide based on playback requirements)
- Tauri command `spotify_authorize()` → opens auth popup
- Tauri command `spotify_get_token()` → stores/refreshes token in secure store (if available) or localStorage
- `src-tauri/src/streaming/spotify.rs` — Spotify Web API client
  - `search(query)` → returns tracks/albums
  - `play_track(track_id, device_id)` → play via Web Playback SDK
  - Fallback to direct URL if Web Playback SDK unavailable
- SvelteKit component: `SpotifyPlayer.svelte` — embed Web Playback SDK, play/pause/next/prev controls

**Acceptance:**
- Spotify login flow works
- Can search and play Spotify tracks from Mercury
- Player controls sync with Spotify app

**Estimated:** 1-2 weeks

### 29.2 — YouTube & SoundCloud Embeds

**Work:**
- YouTube: Detect Spotify/SoundCloud links → convert to YouTube search → embed `<iframe>` player
- SoundCloud: Use SoundCloud Widget API (if available) or direct `<iframe>` embed
- `lib/embeds.ts` — service detection + embed generation
- Fallback: if no embed available, show "Play on [service]" link

**Acceptance:**
- YouTube and SoundCloud tracks embedded and playable
- Multiple service options available for same track
- Graceful fallback if service unavailable

**Estimated:** 1 week

### 29.3 — Bandcamp Integration

**Work:**
- Detect Bandcamp links in artist/release pages
- Embed Bandcamp player (iframe or API, check Bandcamp ToS)
- Wire into track playback selection

**Acceptance:**
- Bandcamp players render inline
- User can play Bandcamp-hosted tracks

**Estimated:** 3-5 days

## Phase 30: Service Preference & Player UI

Add UI to switch between streaming services + persist preference.

### 30.1 — Service Selector

**Work:**
- `StreamingPreference` type: `{ preferred: 'spotify' | 'youtube' | 'soundcloud' | 'bandcamp' | 'auto' }`
- Store in `preferenceStore` (localStorage-backed)
- Player bar: add dropdown/tabs showing available services for current track
- When user selects a service, remember it globally + per-track
- Display which service is currently playing (icon + name)

### 30.2 — Availability Indicator

**Work:**
- On artist/release pages, show which services have this track available
- Badge system: "Play on [service]" badges inline with availability
- Priority order: user preference → available services → fallback link

### 30.3 — Settings Panel

**Work:**
- Settings → Streaming Preferences
- Choose default service (Spotify, YouTube, SoundCloud, Bandcamp, auto-detect)
- Toggle auto-fallback (if primary unavailable, play on next available)

**Acceptance:**
- User preference persists across sessions
- Player respects preference
- Available services visible on every track surface

**Estimated:** 1 week
